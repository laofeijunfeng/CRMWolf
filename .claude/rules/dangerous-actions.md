# CRITICAL 操作阻断清单

**适用范围**：所有 Claude 执行的操作

---

## CRITICAL 级别操作清单

以下操作被标记为 CRITICAL，**必须强制阻断**：

| 操作类型 | 触发条件 | 阻断措施 |
|----------|----------|----------|
| 删除客户 | `delete_customer` intent | Preview + 二次确认 + 审批流 |
| 批量修改 | 批量数量 > 10 | Preview + 二次确认 |
| 合同作废 | 合同状态变更为作废 | Preview + 审批流校验 |
| 修改生产配置 | 涉及 config 文件 | 拒绝执行，需人工审批 |
| 推送 main 分支 | 目标分支为 main/master | 拒绝执行 |

---

## AI 禁止执行的操作

| 操作 | 原因 |
|------|------|
| 直接推送 main/master 分支 | 违反分支纪律 |
| 修改 tsconfig/eslint.config/pyproject.toml | 需人工审批 |
| 修改 Alembic 迁移文件（历史版本） | 需人工审批 |
| 执行未 Preview 的 CRITICAL 操作 | 违反 Preview 强制规则 |
| 绕过 CRUD 直接操作数据库 | 违反 CRUD 统一入口规则 |
| 删除未备份的业务数据 | 数据安全风险 |

---

## 阻断后处理流程

```
触发阻断 → 停止执行 → 填写 COMPLIANCE 报告 → 等待人工审批
```

### COMPLIANCE 报告模板

```markdown
## 合规问题报告

**触发时间**：YYYY-MM-DD HH:MM
**触发操作**：[操作描述]
**阻断原因**：[CRITICAL 操作类型]
**请求内容**：[用户请求原文]
**建议处理**：[人工审批 / 修改请求 / 放弃操作]
```

---

## 二次确认机制

对于需要二次确认的操作，Claude 必须：

1. **展示变更计划**：列出所有受影响的实体和字段
2. **请求显式确认**：用户必须输入"确认"或点击确认按钮
3. **记录确认日志**：将确认行为记录到审计日志

---

## 审批流校验

涉及审批流的操作，Claude 必须校验：

| 校验项 | 要求 |
|--------|------|
| 审批流存在 | 检查业务实体是否配置了审批流 |
| 用户权限 | 检查用户是否有发起审批的权限 |
| 审批状态 | 检查审批流是否已完成 |
