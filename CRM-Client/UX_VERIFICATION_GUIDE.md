# AI Execution Log UX功能验证指引

## 验证准备

启动开发服务器：
```bash
cd CRM-Client
npm run dev
```

访问 AI Assistant页面：`http://localhost:5173/ai-assistant`

---

## Phase 1: UI优化验证

### 1. Signature元素验证
**目标**: 验证 SmartBoundaryLine 流动感渐变线条

**步骤**:
1. 发送任意 AI 指令（如"查找客户光大证券"）
2. 观察 AgentExecutionLog 顶部是否出现**蓝色渐变线条**
3. 线条应有**流动动画**（active状态）
4. 执行完成后，线条应**消失**

**预期**: 流动感渐变线条，仅在执行中显示

---

### 2. 温暖空状态验证
**目标**: 验证 EmptyState 温暖提示 + 首次用户引导

**步骤**:
1. 清空 localStorage（`localStorage.clear()`）
2. 刷新页面，进入 AI Assistant
3. 观察 AgentExecutionLog 显示：
   - **Cpu图标**（24px，蓝色）
   - **温暖文案**："AI 准备就绪，等待你的指令"
   - **首次提示气泡**："💡 输入指令后，AI 的执行过程会在这里实时展示"
4. 点击"知道了"按钮，气泡消失
5. 再次刷新，气泡**不再显示**（已标记已看）

**预期**: 温暖欢迎 + 首次提示，localStorage 存储已看标记

---

### 3. 状态颜色区分验证
**目标**: 验证思考中/执行中/成功/失败 4种状态颜色

**步骤**:
1. 发送 AI 指令："查找客户光大证券"
2. 观察 AgentExecutionLog（收起状态）：
   - **思考中**: 蓝色图标（Cpu）
   - **执行中**: 蓝色旋转图标（Loading）
3. 执行成功后：
   - **成功**: 绿色图标（CircleCheckFilled）
   - 文字颜色：`$wolf-success-text` (#2B633C)
4. 测试失败场景（发送错误指令）：
   - **失败**: 红色图标（CircleCloseFilled）
   - 文字颜色：`$wolf-danger-text` (#7A2828)

**预期**: 颜色 + 图标双重区分，符合 WCAG AA对比度

---

### 4. 进度计数验证
**目标**: 验证 Round N/M 进度显示

**步骤**:
1. 发送多轮 ReAct 指令："查找客户光大证券，然后创建跟进记录"
2. 观察收起状态显示：
   - **Round 1/2 · 正在查询客户信息**
   - **Round 2/2 · 正在创建跟进记录**

**预期**: 进度计数清晰，用户知道剩余轮次

---

### 5. 无障碍验证
**目标**: 验证 WCAG 2.1 Level AA

**步骤**:
1. **键盘导航**: Tab 进入 AgentExecutionLog → Enter 展开 → Escape 收起
2. **Focus状态**: 观察 focus-visible outline（2px蓝色边框）
3. **ARIA属性**: 使用屏幕阅读器验证 aria-label
4. **Reduced Motion**: 系统设置启用减少动画 → 验证旋转动画禁用

**预期**: 键盘可操作，focus可见，屏幕阅读器可宣告，动画可禁用

---

## Phase 3: UX优化验证

### 6. 自动收起验证
**目标**: 验证 3秒自动收起 + 倒计时显示

**步骤**:
1. 发送 AI 指令："查找客户光大证券"
2. AgentExecutionLog 默认**展开**（执行中）
3. 执行完成后，观察：
   - **倒计时提示**："执行完成，3秒后自动收起"
   - **倒计时更新**: 2秒 → 1秒 → 0秒
4. 3秒后，自动**收起**为单行
5. 点击"保持展开"按钮，取消自动收起

**预期**: 3秒倒计时 + 视觉反馈 + 用户可取消

---

### 7. 悬停预览验证
**目标**: 验证悬停 Tooltip 显示进度摘要

**步骤**:
1. 执行完成后，AgentExecutionLog 收起为单行
2. 鼠标**悬停**在收起状态上
3. 观察 Tooltip 显示：
   - **Round 2/5**（进度）
   - **正在查询客户信息**（当前步骤）
   - **执行中**（状态）
   - **执行时长: 15s**（时长）
4. Tooltip 有**提示**："点击查看完整轨迹"

**预期**: 悬停显示进度摘要 + 执行时长

---

### 8. 轨迹导航验证
**目标**: 验证点击步骤跳转到消息 + 高亮

**步骤**:
1. 展开 AgentExecutionLog
2. 点击任意**步骤卡片**（如"查找客户信息"）
3. 观察：
   - 页面**自动滚动**到对应消息
   - 消息**高亮 2秒**（蓝色背景 + 左侧边框）
   - 高亮消失后恢复正常

**预期**: 点击跳转 + 高亮 2秒 + 自动消失

---

## Phase 2: 持久化验证

### 9. 页面刷新恢复验证
**目标**: 验证刷新后 executionSteps恢复

**步骤**:
1. 发送 AI 指令："查找客户光大证券"
2. 等待执行完成
3. **刷新页面**（F5）
4. 观察 AgentExecutionLog：
   - 应显示**历史执行步骤**
   - 可展开查看完整轨迹

**预期**: 刷新后 executionSteps 不丢失，完整恢复

---

### 10. localStorage缓存验证
**目标**: 验证 SSE中断保护

**步骤**:
1. 发送 AI 指令："查找客户光大证券"
2. 执行中途**关闭浏览器标签页**（模拟 SSE中断）
3. 重新打开页面
4. 观察 localStorage：
   - `localStorage.getItem('execution_steps_<conversationId>')`
   - 应有缓存数据
5. 刷新页面，executionSteps 应**从缓存恢复**

**预期**: SSE中断时 localStorage缓存保护数据
