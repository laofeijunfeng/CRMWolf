# CRMWolf 设计原则

⚠️ **品牌色唯一来源**：`src/styles/variables.scss`

**所有颜色必须引用 Sass 变量，禁止硬编码。颜色定义不在本文档中。**

---

## 一、设计原则

### 1.0 Signature 元素（独特视觉标识）

根据 frontend-design skill 要求：每个产品应有"让人记住"的独特元素。

| Signature | 实现 | 位置 |
|-----------|------|------|
| **AI 对话区** | IBM Plex Mono 字体 + 微蓝背景 + 智能边线 | `ChatBubble.vue` |
| **智能边界线** | 流动感渐变线条 | `AIAssistant.vue` 输入区顶部 |
| **不对称圆角** | 用户：左下角直角；AI：右下角直角 | `ChatBubble.vue` |

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **简约至上** | 去除多余装饰，保留核心信息 |
| **层次分明** | 浅灰背景 + 白色卡片建立清晰层级 |
| **轻量高效** | 柔和阴影、细腻圆角、流畅动效 |
| **一致统一** | 全系统使用同一套设计 token（variables.scss） |
| **信息密度** | 在保证可读性的前提下，提高信息密度 |
| **克制 Display font** | IBM Plex Sans 仅用于页面标题（≤5%） |

### 1.2 命名空间

所有 CSS 类名使用 `wolf-` 前缀：

```css
.wolf-card      /* 卡片 */
.wolf-tag       /* 标签 */
.wolf-btn       /* 按钮 */
.wolf-table     /* 表格 */
.wolf-layout    /* 布局 */
.wolf-ai-message /* AI 消息（← Signature） */
```

## 四、交互规范

### 4.1 Cursor 规范

| 元素类型 | Cursor | Sass 变量 |
|----------|--------|-----------|
| 可点击按钮 | `pointer` | `$wolf-cursor-clickable` |
| 可点击卡片 | `pointer` | `$wolf-cursor-clickable` |
| 禁用状态 | `not-allowed` | `$wolf-cursor-disabled` |

### 4.2 Hover 状态规范

| 规则 | 推荐 | 禁止 |
|------|------|------|
| 视觉反馈 | color/opacity/shadow 变化 | transform: scale() |
| 过渡时长 | 150-300ms | >500ms 或 0ms |
| 布局影响 | 无位移 | 任何导致布局重排的变化 |

### 4.3 Focus 状态规范

| 元素 | Focus 样式 | Sass 变量 |
|------|-----------|-----------|
| 所有可交互元素 | `box-shadow: 0 0 0 2px rgba($wolf-primary, 0.3)` | `$wolf-focus-ring-color` |

**应用方式**：使用 `*:focus-visible` 全局选择器

### 4.4 条件联动原则（新增）

**适用场景**：开关、前置条件与下游元素的联动交互。

| 原则 | 规则 | 示例 |
|------|------|------|
| **开关联动** | 开关关闭时，下游字段应禁用（灰色背景）+ 可见，并显示警告提示 | 启用通知关闭 → Webhook URL 字段禁用 + "请先开启开关"提示 |
| **禁用原因告知** | 按钮/元素禁用时，必须告知用户原因（tooltip 或提示条） | 测试按钮禁用 → tooltip 显示"请先填写 Webhook URL" |
| **数据保留** | 开关关闭不清空已填数据，重新开启后数据恢复 | 关闭通知 → URL 保留；重新开启 → URL 自动恢复 |

**视觉表现**：

```scss
// 禁用字段样式
.input-disabled {
  background: $wolf-bg-hover;           // 灰色背景
  cursor: not-allowed;
  opacity: 0.6;
}

// 警告提示（黄色）
.form-tip--warning {
  color: $wolf-warning-text;
  display: flex;
  align-items: center;
  gap: $wolf-space-xs;
}
```

**实现要点**：

```vue
<!-- 开关联动示例 -->
<el-form-item label="Webhook URL">
  <el-input
    v-model="formData.url"
    :disabled="!formData.enabled"
  />
  <div v-if="!formData.enabled" class="form-tip form-tip--warning">
    <el-icon><Warning /></el-icon>
    <span>请先开启开关</span>
  </div>
</el-form-item>
```

### 4.5 状态告知原则（新增）

**适用场景**：操作成功/失败、按钮禁用等状态反馈。

| 原则 | 规则 | 示例 |
|------|------|------|
| **成功后下一步** | 操作成功后，提示用户下一步可做什么 | 测试成功 → "配置生效后，审批提交时会自动通知此群" |
| **失败后建议** | 操作失败后，列出可能原因和解决方案 | 测试失败 → "可能原因：URL 不正确/机器人被移除/网络问题" |
| **禁用前置检查** | 禁用按钮上方显示前置条件提示条 | 测试按钮禁用 → 上方黄色提示条 "请先填写 Webhook URL" |

**状态矩阵示例**：

| 条件 | 按钮状态 | Tooltip/提示内容 |
|------|----------|------------------|
| 前置条件 A 未满足 | 禁用 | "请先完成前置条件 A" |
| 前置条件 B 未满足 | 禁用 | "请先完成前置条件 B" |
| 所有条件满足 | 可点击 | 无 tooltip |

### 4.6 操作引导原则（新增）

**适用场景**：新用户首次操作、复杂配置流程。

| 原则 | 规则 | 示例 |
|------|------|------|
| **步骤式引导** | 新用户首次配置应有清晰的步骤引导（数字标记 + 标题 + 操作） | ①添加机器人 ②复制URL ③粘贴测试 |
| **可复制示例** | 配置示例值可一键复制，降低输入成本 | Webhook URL 示例 → 点击复制到剪贴板 |
| **官方文档链接** | 提供官方文档链接，帮助用户理解概念 | "飞书群聊机器人配置指南" 链接 |

**视觉设计**：

```
┌─────┐  1. 步骤标题
│  1  │     → 操作链接/可复制示例
└─────┘
┌─────┐  2. 步骤标题
│  2  │     → 操作提示
└─────┘
```

**实现要点**：

```vue
<div class="guide-step">
  <div class="step-number">1</div>
  <div class="step-content">
    <div class="step-title">在飞书群聊中添加机器人</div>
    <a href="..." target="_blank">查看官方文档 →</a>
  </div>
</div>
```

### 4.7 保存-验证联动原则（新增）

**适用场景**：配置保存后需要验证的场景。

| 原则 | 规则 | 示例 |
|------|------|------|
| **保存后询问** | 保存成功后，如满足验证条件，询问是否立即验证 | "配置已保存，是否立即发送测试消息验证？" |
| **减少操作** | 用户可选择"测试"立即验证，或"稍后"跳过 | 确认框：[发送测试] [稍后验证] |

**实现要点**：

```typescript
const handleSave = async () => {
  await saveConfig()
  showSuccess('保存', '配置')
  
  // 保存成功后询问是否验证
  if (canTest) {
    ElMessageBox.confirm(
      '配置已保存，是否立即验证？',
      '验证配置',
      { confirmButtonText: '验证', cancelButtonText: '稍后' }
    ).then(() => handleTest()).catch(() => {})
  }
}
```

## 五、无障碍规范

### 5.1 Reduced Motion

```scss
@media (prefers-reduced-motion: reduce) {
  * { transition-duration: 0.01ms !important; }
}
```

**必须遵守**：所有动画/过渡需尊重用户偏好

### 5.2 对比度要求（WCAG）

| 颜色角色 | 最小对比度 | 验证状态 |
|----------|-----------|----------|
| 正文文字 | 4.5:1 (AA) | ✅ 已合规 |
| Placeholder | 4.5:1 (AA) | ✅ 已合规 (#7A7A7A) |
| 禁用态 | 不要求 | ✅ 允许低于标准 |

## 六、设计红线

| 红线 | 说明 |
|------|------|
| 禁止硬编码颜色 | 必须引用 variables.scss 变量 |
| 禁止纯色填充标签 | 必须用浅底色+同色系文字 |
| 禁用 700+ 字重 | 最大使用 semibold (600) |
| 禁止高饱和色 | 使用 variables.scss 低饱和色系 |
| 禁止多层阴影 | 卡片仅用一级阴影 |
| 禁止魔数间距 | 向 4px 基准网格对齐 |
| **禁止卡片用 Display font** | 卡片标题使用系统字体栈 |
| **禁止缺少 focus 状态** | 可交互元素必须有可见 focus |
| **禁止忽略 reduced motion** | 动画必须尊重用户偏好 |
| **禁止禁用无提示** | 按钮/元素禁用时必须告知原因 |
| **禁止开关无联动** | 开关关闭时下游字段应禁用+提示 |
| **禁止使用 emoji 作为图标** | 使用 Element Plus/SVG 图标组件 |
| **禁止成功无下一步** | 操作成功后必须提示下一步操作 |

## 七、间距命名规范

**仅使用语义化命名**，废弃数字命名：

| 变量 | 值 | 用途 |
|------|-----|------|
| `$wolf-space-xs` | 4px | 元素内间距（图标与文字） |
| `$wolf-space-sm` | 8px | 关联元素间距 |
| `$wolf-space-md` | 16px | 模块内间距（卡片内边距） |
| `$wolf-space-lg` | 24px | 模块间间距、页面安全边距 |

❌ **废弃**：`$wolf-space-1/2/3/4/5/6/8/10/12`（数字命名）
