## 2. 线索管理

### 2.1 功能概述

线索管理模块负责销售线索的全生命周期管理，包括线索录入、跟进、判定、转化等功能。

### 2.2 线索状态

| 状态 | 值 | 说明 |
|------|----|------|
| NEW | 0 | 新线索 |
| FOLLOWING | 1 | 跟进中 |
| CONVERTED | 2 | 已转化 |
| INVALID | 3 | 无效线索 |

### 2.3 线索来源

| 来源 | 说明 |
|------|------|
| 线上注册 | 官网注册 |
| 市场活动 | 展会、沙龙等 |
| 客户推荐 | 老客户转介绍 |
| 电话营销 | 电话开发 |
| 网站咨询 | 官网咨询 |
| 展会 | 行业展会 |
| 其他 | 其他渠道 |

### 2.4 线索转化流程

```
┌─────────────────────────────────────────────────────────────┐
│                      线索转化流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  新线索 ──▶ 跟进联系 ──▶ 线索判定                           │
│                              │                              │
│              ┌───────────────┼───────────────┐              │
│              │               │               │              │
│              ▼               ▼               ▼              │
│         无效线索         继续跟进         合格转化          │
│         (关闭)          (跟进中)     (创建客户+商机)        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.5 公海池

- 公海线索：无负责人的线索
- 认领机制：销售人员可从公海认领线索
- 退回机制：跟进不力的线索可退回公海

## 3. 客户管理

### 3.1 功能概述

客户管理模块管理企业客户档案、联系人信息、跟进记录等。

### 3.2 客户状态

| 状态 | 值 | 说明 |
|------|----|------|
| FOLLOWING | 0 | 跟进中 |
| WON | 1 | 已赢单 |
| LOST | 2 | 已输单 |
| INACTIVE | 3 | 已失效 |

### 3.3 客户信息

- **基本信息**：公司名称、行业、城市、地址、规模
- **联系人**：姓名、职位、电话、邮箱、是否决策人
- **跟进记录**：跟进内容、跟进方式、下次跟进时间

### 3.4 数据权限

| 角色 | 数据范围 |
|------|----------|
| 销售成员 | 仅查看自己负责的客户 |
| 销售总监 | 查看所有客户 |
| 财务 | 查看所有客户（仅发票相关） |
| 系统管理员 | 查看所有客户 |

## 4. 商机管理

### 4.1 功能概述

商机管理模块管理销售机会的阶段推进、金额预测、赢单/输单判定。

### 4.2 商机状态

| 状态 | 值 | 说明 |
|------|----|------|
| FOLLOWING | 0 | 跟进中 |
| WON | 1 | 已赢单 |
| LOST | 2 | 已输单 |

### 4.3 销售阶段

| 阶段 | 赢率 | 说明 |
|------|------|------|
| 初步接洽 | 10% | 初次接触客户 |
| 需求确认 | 20% | 明确客户需求 |
| 方案报价 | 40% | 提供解决方案 |
| 商务谈判 | 60% | 价格和条款谈判 |
| 签订合同 | 90% | 准备签约 |
| 赢单 | 100% | 成交 |

### 4.4 商机推进流程

```
初步接洽 ──▶ 需求确认 ──▶ 方案报价 ──▶ 商务谈判 ──▶ 签订合同 ──▶ 赢单
                                                            │
                              ┌───────────────────────────────┘
                              │
                              ▼
                           输单
```

### 4.5 采购类型

| 类型 | 说明 |
|------|------|
| NEW | 新购 |
| RENEWAL | 续购 |
| EXPANSION | 增购 |

### 4.6 授权模式

| 模式 | 说明 |
|------|------|
| SUBSCRIPTION | 订阅制（按年付费） |
| PERPETUAL | 买断制（一次性付费） |

## 5. 合同管理

### 5.1 功能概述

合同管理模块负责合同的创建、审批、执行跟踪。

### 5.2 合同状态

| 状态 | 说明 |
|------|------|
| DRAFT | 草稿 |
| PENDING_REVIEW | 待审批 |
| SIGNED | 已签订 |
| EFFECTIVE | 生效中 |
| EXPIRED | 已到期 |
| TERMINATED | 已终止 |

### 5.3 回款状态

| 状态 | 说明 |
|------|------|
| UNPAID | 未回款 |
| PARTIAL | 部分回款 |
| COMPLETED | 已完成 |
| OVERDUE | 已逾期 |

### 5.4 合同审批流程

```
┌─────────────────────────────────────────────────────────────┐
│                      合同审批流程                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  草稿 ──▶ 待审批 ──▶ 已签订 ──▶ 生效中 ──▶ 已到期          │

│               │                                             │
│               └──▶ 修改后重新提交                           │

```

### 5.5 操作权限

| 操作 | 草稿 | 待审批 | 已签订 | 已拒绝 |

| 删除 | ✓ | ✗ | ✗ | ✓ |
| 提交审批 | ✓ | ✗ | ✗ | ✓ |
| 撤回 | ✗ | ✓ | ✗ | ✗ |
