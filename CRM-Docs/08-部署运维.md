## 3. 后端部署

### 3.1 安装依赖

```bash
# 安装 Python 3.11
sudo apt update
sudo apt install python3.11 python3.11-venv python3-pip

# 创建应用目录
sudo mkdir -p /opt/crmwolf
sudo chown $USER:$USER /opt/crmwolf
```

### 3.2 部署代码

```bash
# 克隆代码
cd /opt/crmwolf
git clone <repository_url> .

# 创建虚拟环境
cd CRM-Server
python3.11 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 文件，配置生产环境参数
```

### 3.3 配置 Systemd 服务

创建服务文件 `/etc/systemd/system/crmwolf-api.service`：

```ini
[Unit]
Description=CRMWolf API Server
After=network.target

[Service]
Type=simple
User=www-data
Group=www-data
WorkingDirectory=/opt/crmwolf/CRM-Server
Environment="PATH=/opt/crmwolf/CRM-Server/venv/bin"
ExecStart=/opt/crmwolf/CRM-Server/venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000 --workers 4
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
sudo systemctl daemon-reload
sudo systemctl enable crmwolf-api
sudo systemctl start crmwolf-api
sudo systemctl status crmwolf-api
```

## 5. Nginx 配置

### 5.1 安装 Nginx

```bash
sudo apt install nginx
```

### 5.2 配置虚拟主机

创建配置文件 `/etc/nginx/sites-available/crmwolf`：

```nginx
server {
    listen 80;
    server_name crm.example.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name crm.example.com;

    # SSL 配置
    ssl_certificate /etc/letsencrypt/live/crm.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/crm.example.com/privkey.pem;

    # 前端静态文件
    location / {
        root /var/www/crmwolf;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # API 文档
    location /docs {
        proxy_pass http://127.0.0.1:8000;
    }

    location /redoc {
        proxy_pass http://127.0.0.1:8000;
    }

    location /openapi.json {
        proxy_pass http://127.0.0.1:8000;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/crmwolf /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 6. 数据库配置

### 6.1 MySQL 配置

```bash
# 安装 MySQL
sudo apt install mysql-server

# 安全配置
sudo mysql_secure_installation

# 创建数据库和用户
mysql -u root -p
```

```sql
CREATE DATABASE crm_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'crm'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON crm_db.* TO 'crm'@'localhost';
FLUSH PRIVILEGES;
```

### 6.2 Redis 配置

```bash
# 安装 Redis
sudo apt install redis-server

# 配置 Redis
sudo nano /etc/redis/redis.conf
```

关键配置：
```
bind 127.0.0.1
requirepass your_redis_password
```

```bash
sudo systemctl restart redis-server
```

## 9. 备份策略

### 9.1 数据库备份

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup/mysql"
MYSQL_USER="crm"
MYSQL_PASSWORD="your_password"
DATABASE="crm_db"

mkdir -p $BACKUP_DIR

mysqldump -u$MYSQL_USER -p$MYSQL_PASSWORD $DATABASE | gzip > $BACKUP_DIR/crm_db_$DATE.sql.gz

# 删除30天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete
```

### 9.2 定时备份

```bash
# 添加 crontab
crontab -e

# 每天凌晨2点备份
0 2 * * * /opt/crmwolf/scripts/backup.sh
```

## 10. 故障排查

### 10.1 服务无法启动

```bash
# 检查服务状态
sudo systemctl status crmwolf-api

# 检查日志
sudo journalctl -u crmwolf-api -n 100
```

### 10.2 数据库连接失败

```bash
# 检查 MySQL 状态
sudo systemctl status mysql

# 测试连接
mysql -u crm -p -h localhost crm_db
```

### 10.3 Redis 连接失败

```bash
# 检查 Redis 状态
sudo systemctl status redis-server

# 测试连接
redis-cli ping
```
