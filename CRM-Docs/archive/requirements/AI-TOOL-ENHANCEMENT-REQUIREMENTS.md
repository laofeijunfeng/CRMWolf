# CRMWolf AI 工具增强需求文档

> **状态：✅ 已实现** | 完成日期：2026-06-08
> 版本：1.0 | 创建日期：2026-06-08

---

## 一、背景与核心痛点

### 1.1 现状

CRMWolf 已实现 Function Calling 架构（`ai_tool_service.py`），支持 AI 通过工具调用执行业务操作：
- 线索管理：9 个工具（创建、查询、跟进、转化等）
- 客户管理：4 个工具（创建、查询、跟进）
- 商机管理：8 个工具（创建、查询、推进、赢单/输单等）

### 1.2 核心痛点

| 痛点 | 表现 | 影响 |
|------|------|------|
| **Tools 不完整** | 合同、回款、发票模块无工具 | AI 无法处理"签合同"后的完整业务链路 |
| **只执行第一个工具** | `tool_calls[0]` 只取第一个 | 用户说"签合同"，AI 只创建跟进，不标记赢单、不创建合同 |
| **无 ReAct 循环** | 单轮调用，无法根据执行结果继续 | 有依赖的操作（创建合同需要先赢单）无法自动串联 |
| **实体歧义无处理** | 客户有 2 个商机时无法消解 | 用户说"签合同"，AI 不知道是哪个商机 |

### 1.3 典型场景举例

**场景 A：签订合同（完整链路）**

用户输入：`"今天已签订合同，30万，50人1年订阅"`

| 期望行为 | 当前行为 | 缺失 |
|----------|----------|------|
| 1. 创建跟进记录 | ✅ 执行 | - |
| 2. 标记商机赢单 | ❌ 不执行 | 只取第一个工具 |
| 3. 创建合同记录 | ❌ 无工具 | 合同 Tools 缺失 |

**场景 B：实体歧义消解**

用户输入：`"已和客户签订合同"`（客户有 2 个商机）

| 期望行为 | 当前行为 | 缺失 |
|----------|----------|------|
| 1. AI 发现客户有多个商机 | ❌ 无此逻辑 | 无歧义检测 |
| 2. 返回需要用户选择的信息 | ❌ 无此机制 | 无 Disambiguation 流程 |
| 3. 用户选择后继续执行 | ❌ 无此机制 | 无上下文继承 |

## 二、需求目标与设计原则

### 2.1 核心目标

1. **补充完整 Tools**：合同、回款、发票模块的工具定义
2. **支持 ReAct 循环**：多轮 Reasoning → Action → Observation → Repeat
3. **支持实体歧义消解**：当存在多个候选实体时，让用户选择

### 2.2 设计原则

| 原则 | 说明 |
|------|------|
| **AI 决定调用链** | 不硬编码规则，让 AI 根据用户描述自动判断需要哪些工具 |
| **依次确认执行** | 每个工具调用前让用户确认，用户可接受/跳过/修改 |
| **执行结果反馈** | 每轮执行结果反馈给 AI，AI 决定是否需要后续操作 |
| **歧义主动询问** | 当实体不唯一时，AI 返回候选列表让用户选择 |
| **配置开关降级** | 通过开关控制 ReAct 启用，可降级到原有单工具模式 |

### 3.2 Phase 2：ReAct 循环机制

#### 3.2.1 核心流程

```
┌─────────────────────────────────────────────────────────┐
│  用户输入："今天签了合同，30万，50人1年订阅"              │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  Round 1: AI 分析                                        │
│  返回 tool_calls = [                                     │
│    {name: "follow_up_customer", params: {...}},          │
│    {name: "win_opportunity", params: {...}}              │
│  ]                                                       │
│  注：create_contract 需等赢单成功                        │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  用户依次确认 → 执行 → 收集结果                          │
│  Observation:                                            │
│    - follow_up_customer: 成功                            │
│    - win_opportunity: 成功，商机 ID=456                  │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  Round 2: AI 收到结果，继续分析                          │
│  返回 tool_calls = [                                     │
│    {name: "create_contract", params: {                   │
│      opportunity_id: 456, total_amount: 300000, ...      │
│    }}                                                    │
│  ]                                                       │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  用户确认 → 执行                                         │
│  Observation: create_contract: 成功，合同 ID=789        │
└──────────────────────────┬──────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  Round 3: AI 判断完成                                    │
│  返回："已完成跟进记录、商机赢单、合同创建"               │
└─────────────────────────────────────────────────────────┘
```

#### 3.2.2 消息格式（OpenAI 标准）

```python
messages = [
    {"role": "user", "content": "今天签了合同..."},
    
    # Round 1: AI 返回工具调用
    {"role": "assistant", "tool_calls": [
        {"id": "call_1", "function": {"name": "follow_up_customer", "arguments": "..."}},
        {"id": "call_2", "function": {"name": "win_opportunity", "arguments": "..."}}
    ]},
    
    # 用户确认后执行，返回结果
    {"role": "tool", "tool_call_id": "call_1", "content": "跟进记录创建成功"},
    {"role": "tool", "tool_call_id": "call_2", "content": "商机赢单成功，商机ID=456"},
    
    # Round 2: AI 继续分析
    {"role": "assistant", "tool_calls": [
        {"id": "call_3", "function": {"name": "create_contract", "arguments": "..."}}
    ]},
    
    # 执行结果
    {"role": "tool", "tool_call_id": "call_3", "content": "合同创建成功"},
    
    # Round 3: AI 判断完成
    {"role": "assistant", "content": "已完成所有操作"}
]
```

#### 3.2.3 安全控制

| 控制项 | 配置 |
|--------|------|
| 最大轮数 | `REACT_MAX_ROUNDS = 5`，防止无限循环 |
| 单轮超时 | `REACT_SINGLE_ROUND_TIMEOUT = 30s` |
| 总超时 | `REACT_TOTAL_TIMEOUT = 120s` |
| 开关控制 | `REACT_ENABLED = False`（默认关闭，稳定后开启） |

### 3.3 Phase 3：实体歧义消解

#### 3.3.1 场景定义

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| **商机歧义** | 客户有多个跟进中商机 | 返回候选列表，让用户选择 |
| **联系人歧义** | 客户有多个联系人 | 返回候选列表，让用户选择 |
| **合同歧义** | 客户有多份合同 | 返回候选列表，让用户选择 |

#### 3.3.2 交互流程

```
用户："已和客户签订合同"

AI 分析：
  - 客户 ID 已确定（从上下文或名称提取）
  - 查询该客户的商机列表 → 发现 2 个跟进中商机
  
AI 返回：
{
  "event": "disambiguation_required",
  "entity_type": "opportunity",
  "candidates": [
    {"id": 101, "name": "CRM项目-50人1年订阅", "amount": 300000},
    {"id": 102, "name": "OA项目-30人1年订阅", "amount": 180000}
  ],
  "message": "该客户有多个商机，请选择签订合同的是哪个？"
}

前端展示选择界面：
┌──────────────────────────────────────┐
│ 请选择签订合同的商机：                │
│                                      │
│ ○ CRM项目-50人1年订阅 (30万)         │
│ ○ OA项目-30人1年订阅 (18万)          │
│                                      │
│ [确认] [取消]                         │
└──────────────────────────────────────┘

用户选择后，将选择结果加入上下文：
messages.append({
  "role": "user", 
  "content": "是 CRM项目那个商机"
})

AI 继续分析，此时 opportunity_id 已确定，继续调用工具
```

#### 3.3.3 后端实现要点

```python
# 在 handle_message_stream 中增加歧义检测

async def handle_message_stream(...):
    # ... AI 返回 tool_calls
    
    for call in tool_calls:
        # 检测实体歧义
        if self._has_entity_ambiguity(call, db, team_id):
            candidates = self._get_entity_candidates(call, db, team_id)
            yield {
                "event": "disambiguation_required",
                "entity_type": call["entity_type"],
                "candidates": candidates,
                "tool_call": call  # 保存，等待用户选择后继续
            }
            return  # 暂停，等待用户选择
    
    # 无歧义，正常执行
    for call in tool_calls:
        yield {"event": "parsed", "tool": call["name"], "params": call["params"]}
```

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 单轮 AI 响应时间 | ≤ 5s |
| ReAct 总响应时间 | ≤ 120s |
| 工具执行时间 | ≤ 3s/个 |

### 4.2 可用性要求

| 指标 | 要求 |
|------|------|
| ReAct 开关降级 | 支持配置开关，关闭时降级到原有单工具模式 |
| 错误隔离 | 单个工具失败不影响其他工具执行 |
| 日志追踪 | 每轮完整记录，支持排查 |

### 4.3 兼容性要求

| 要求 | 说明 |
|------|------|
| 向后兼容 | 不影响现有单工具调用逻辑 |
| API 格式兼容 | SSE 流式响应格式保持不变 |
| 前端兼容 | 新增事件不影响原有事件处理 |

## 六、风险评估与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| ReAct 无限循环 | 资源耗尽 | max_rounds=5 + 总超时 120s |
| AI 返回错误工具 | 业务数据错误 | 风险分级 + 强制确认 |
| 实体歧义频繁触发 | 用户体验差 | 置信度阈值 + 智能排序 |
| 前端改动大 | 开发周期长 | 分阶段实施 + 配置开关 |
