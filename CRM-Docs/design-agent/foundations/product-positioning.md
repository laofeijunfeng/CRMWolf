# 产品定位

- **用途：**定义 CRM AI Agent 的产品目标和业务范围。
- **适用范围：**Agent 需求、交互、Prompt、Tool 和后端实现。
- **权威性：**本文件拥有 Agent 产品定位规则。
- **相关规范：**[架构边界](architecture-boundary.md) · [Tool 规范](../runtime/tools.md)

## 核心定位

Agent 是围绕客户跟进记录的智能客户关系管理系统能力，不是通用聊天助手。

它的首要目标是让销售更方便地录入高质量客户跟进记录，并基于跟进内容和客户上下文，优先协助创建商机、补齐商机信息和推动商机流程。

合同、回款、发票和 License 属于后续链路。当前阶段 Agent 可以读取上下文、说明缺失环节、给出下一步建议；除非 tool 范围明确纳入，否则不进入自动写入闭环。

## 核心方向

- 降低销售录入跟进记录的操作成本。
- 用 AI 结构化理解跟进内容、客户、下一步动作和时间。
- 基于现有 CRM API 查询客户上下文。
- 基于上下文生成可解释的下一步建议。
- 在用户确认后执行写入动作。

## 优先业务场景

- 客户跟进记录创建。
- 基于跟进记录识别商机信号、创建商机和推动商机。
- 客户联系人、发票抬头、部署信息等客户基础信息维护。
- 销售管理者查询客户情况、商机进展和团队经营摘要。
- 回款、发票和 License 后续作为建议或查询场景，不作为当前主闭环。

## 主业务链路

CRM 主业务链路是：客户 -> 商机 -> 合同 -> 回款计划 -> 登记回款 -> 发票 -> License 申请。

Agent 生成建议或执行动作时必须尊重链路前置条件：

- 没有商机和合同时，回款场景应优先引导补齐商机。
- 有商机但没有合同时，应说明合同环节缺失；当前不建议创建回款计划或登记回款。
- 创建回款计划必须依赖已有明确合同。
- 登记回款必须依赖已有明确未完成回款计划。

当前阶段商机是 Agent 主业务推进边界：

- 跟进内容出现立项、预算、人数、采购方式、招标、试用转正式等信号时，优先判断商机创建或商机推进。
- 商机信息缺失时，只追问系统必填且无法可靠推断的字段。
- 合同及后续动作优先提示用户回到系统查看和处理。

License 申请存在分支前置条件：

- 试用 License 可以在商机审批通过后申请。
- 正式 License 必须在合同审批通过后申请。
- 两类 License 都必须先确认部署信息。

## 暂不支持

- 创建合同，因为现有创建合同流程需要上传合同附件。
- 绕过审批流程直接创建或提交业务对象。
- 在没有客户权限或上下文的情况下补造业务信息。
