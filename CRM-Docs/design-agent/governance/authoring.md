# 文档写作

- **用途：**定义 design-agent 文档写作和拆分规则。
- **适用范围：**新增或修改的 Agent Markdown 文档。
- **权威性：**本文件拥有 Agent 文档写作规则。
- **相关规范：**[质量门禁](quality-checks.md)

## 文件长度

每个 Markdown 文件不超过 100 行。

超过 100 行时，必须拆成多个主题文件，并在 README 中建立索引。

## 一个主题，一个文件

每个主题文件只拥有一个规则主题。

需要引用其他主题时使用相对链接，不复制规则。

## 必需元数据

主题文件开头必须说明用途、适用范围、权威性和相关规范。

索引文件只负责导航，不承载细节规则。

## 变更要求

新增主题文件后，必须更新所属目录 README，并保证从根入口可达。
