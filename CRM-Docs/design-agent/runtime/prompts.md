# Prompt 规范

- **用途：**定义 CRM AI Agent Prompt 的职责和边界。
- **适用范围：**语义解析、业务建议、tool-calling 子 Agent。
- **权威性：**本文件拥有 Agent Prompt 规则。
- **相关规范：**[结构化输出](structured-output.md) · [交互编排策略](interaction-policy.md) · [Tool 规范](tools.md)

## 基本要求

Prompt 必须结合 CRMWolf 业务，不允许使用通用客服或通用助手模板。

每个 Prompt 必须明确模型职责：语义解析只理解输入，业务建议只生成建议，tool-calling 子 Agent 只在授权 tool 集内规划。

Prompt 可以要求模型输出候选动作，但不得要求模型直接决定最终用户交互顺序；最终顺序由 interaction planner 按策略裁决。

## 业务链路提示

业务建议 Prompt 必须显式包含 CRM 主链路：客户 -> 商机 -> 合同 -> 回款计划 -> 登记回款 -> 发票 -> License 申请。

当前阶段 Prompt 必须强调 Agent 主边界：

- 跟进记录是首要产物。
- 商机创建、商机字段补充和商机推进是当前主要业务推进动作。
- 合同、回款、发票和 License 优先作为上下文查询、风险说明和系统操作建议。
- 不得因为用户提到回款或发票，就跳过商机和合同前置条件。

跟进记录 Prompt 必须识别：

- 客户沟通事实、客户反馈、项目阶段和采购信号。
- 预算、人数、授权模式、采购类型、预计成交日期。
- 下一步动作、责任方和时间表达。
- 是否建议创建商机或补齐商机。
- 已有进行中商机时，是否可能涉及推进商机阶段。
- 商机名称不是语义必填项，由后端 API 统一生成，Prompt 不得要求模型生成或追问。
- 系统没有跟进提醒 tool；下一步时间只写入跟进记录，不生成提醒建议。

商机阶段 Prompt 必须遵守：

- 采购阶段以 CRM API 返回的 `procurement_stages` 为准。
- 不得硬编码阶段名称，不得按模型常识创造阶段。
- 多个进行中商机时先追问选择商机。
- 只对审批通过、进行中的商机建议推进阶段。
- 目标阶段必须来自同一商机的阶段列表，且不能是当前阶段。
- 跟进记录仍是首要产物；阶段推进只能作为后续建议和确认动作。

回款相关 Prompt 必须写清：

- 创建回款计划依赖已有明确合同。
- 登记回款依赖已有明确未完成回款计划。
- 没有商机和合同时，优先建议补充商机。
- 有商机但没有合同时，只说明合同环节缺失；创建合同当前不接入 Agent。

License Prompt 必须区分：

- 试用 License：依赖审批通过的商机。
- 正式 License：依赖审批通过且可用的合同。
- 用户未说明试用或正式时，先追问 License 类型。
- 缺少部署信息时，先建议创建或确认部署信息。

## 必含上下文

- 当前系统日期和时区。
- 会话记忆摘要。
- 用户原始输入。
- 已解析结构化语义。
- 通过 CRM API 获取的客户上下文。
- 当前允许的 tool 名称和写入确认状态。

## 禁止事项

- 禁止要求模型输出 Markdown 或解释性自然语言作为结构化结果。
- 禁止要求模型编造 CRM 对象。
- 禁止要求模型判断用户权限。
- 禁止让模型自行执行写入动作。
- 禁止建议当前 tool 清单不存在的能力，例如设置跟进提醒。
- 禁止让模型在最终自然语言中同时追问多个用户响应动作。

## 时间表达

模型只抽取时间语义槽位，例如“下周三”对应相对星期表达。

最终 ISO 日期由代码基于系统日期、时区和业务规则计算。
