# 结构化输出

- **用途：**定义 AI 结构化结果的生产级要求。
- **适用范围：**语义解析、跟进质量评估、业务建议、后续 tool-calling 计划。
- **权威性：**本文件拥有结构化输出规则。
- **相关规范：**[Prompt 规范](prompts.md) · [交互编排策略](interaction-policy.md) · [观测与评估](observability.md)

## 必须结构化的内容

- 用户意图和置信度。
- 客户名称、客户置信度和澄清问题。
- 跟进内容、下一步动作和原文时间表达。
- 跟进质量总分、六大原则分项、补充问题和是否通过。
- 时间语义槽位，不包含模型自行换算的最终日期。
- 回款金额、回款日期、付款方和缺失字段。
- 建议动作、原因、风险、缺失字段和关联对象。
- 候选交互动作、阻塞原因、优先级和是否需要用户响应。

## 校验要求

结构化结果必须通过 Pydantic schema 校验。

置信度不足、客户不明确、字段冲突时，必须进入澄清流程，不得继续执行 tool。

结构化输出可以包含多个候选动作，但最终用户回复只能由 interaction planner 选择 1 个动作展示。

## Fallback 要求

优先使用 LangChain structured output。

兼容性 fallback 可以保留，但必须记录来源、模型、错误原因和校验结果，不得静默降级。

## 前端可见性

调试事件应能展示解析来源、模型名称和是否 fallback，帮助用户判断“是否真的走 AI”。
