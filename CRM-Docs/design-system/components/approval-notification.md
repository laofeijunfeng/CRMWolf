# 审批通知

- **用途：**提供审批待办入口、待办数量和受限的上下文跳转。
- **适用范围：**components 领域。
- **权威性：**本主题是审批通知入口、深链与待办计数语义的唯一事实来源。
- **相关规范：**[组件索引](README.md) · [顶部栏](topbar.md) · [颜色令牌](../foundations/color-tokens.md) · [动效与性能](../foundations/motion-performance.md) · [无障碍](../foundations/accessibility.md)

## 入口与目的地

桌面端审批入口仅为顶部栏中的铃铛按钮；不得在侧栏、顶部栏或其他全局区域提供第二个并列审批入口。铃铛进入审批中心的待办视图，并清楚说明是否存在待处理工作。

通知列表中的上下文深链仅可进入对应审批任务、该任务关联的业务记录详情，或该记录内的审批历史。深链必须保留足以识别任务和返回审批中心的上下文；不得借通知入口跳转到无关模块、编辑页面或未经授权的记录。

## 待办数量语义

待办数量表示当前用户可执行的未完成审批工作数量，不表示已读通知数，也不会因为访问铃铛、打开列表或查看审批中心而归零。无待办时不显示数量提示，但入口仍保留。

数量在批准、驳回、转交、撤回或其他会改变当前用户待办归属或完成状态的操作确认后更新。操作失败或仍在处理中时保留原数量，并提供结果反馈；跨会话或外部变更到达时同步为最新可执行待办。

## 可达性与反馈

铃铛是有可读名称的图标按钮，数量具有可读的待办含义，而不只是一枚视觉圆点。动态数量变化以适当方式通知，不打断正在进行的操作。颜色遵循[颜色令牌](../foundations/color-tokens.md)，焦点、图标名称和动态内容遵循[无障碍](../foundations/accessibility.md)，状态变化与减少动效遵循[动效与性能](../foundations/motion-performance.md)。
