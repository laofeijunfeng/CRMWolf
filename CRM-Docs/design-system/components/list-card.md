# 列表卡片

- **用途：**展示紧凑、可操作的同类型列表项，用于 Sheet、侧栏等受限空间。
- **适用范围：**components 领域。
- **权威性：**本主题是列表卡片的布局、交互与与其他组件选择的唯一事实来源。
- **相关规范：**[组件索引](README.md) · [表格](table.md) · [卡片](card.md) · [间距与布局](../foundations/spacing-layout.md) · [颜色令牌](../foundations/color-tokens.md) · [动效与性能](../foundations/motion-performance.md) · [无障碍](../foundations/accessibility.md)

## 结构与内容

列表卡片承载一个标题和一组同类型列表项。标题说明内容类型，列表项呈现必要信息，操作集中在行内或标题栏。不用于跨行比较，仅用于快速浏览和操作。

同一区域的列表卡片采用一致的信息顺序和间距。列表项内的交互元素保留自身语义；不得让整行与其中的链接、按钮产生相互冲突的点击目标。

## 组件选择：列表卡片 vs 表格

| 维度 | 使用列表卡片 | 使用表格 |
|------|------------|---------|
| **数据量** | ≤50 条 | >50 条需要分页 |
| **交互** | 无需排序、筛选 | 排序、筛选是关键功能 |
| **空间** | Sheet、侧栏等受限空间 | 完整页面列表视图 |
| **操作** | 每行操作按钮（编辑、删除） | 批量操作、行选择 |
| **滚动** | 内容自然滚动，无分页 | 垂直滚动 + 固定分页 |
| **比较** | 不需要跨行比较 | 需要跨行比较数据 |

**选择原则：**
- Sheet 内的实体列表（联系人、商机、合同等）优先使用列表卡片
- 完整列表页、需要复杂查询筛选的场景使用表格
- 当数据量超过 50 条时，评估是否需要切换为表格或增加分页

## 布局与层级

列表卡片使用[卡片](card.md)的基础层级：白色背景、圆角、轻量边框。标题栏与列表项之间以分隔线区分。

标题栏包含标题文字和操作按钮（如"新建"），高度遵循[间距与布局](../foundations/spacing-layout.md)。列表项最小高度 44px 以满足触摸目标要求，内边距与标题栏保持一致节奏。

列表项之间以轻量分隔线组织，不使用斑马纹背景。悬停仅用于可点击项的视觉反馈，使用[颜色令牌](../foundations/color-tokens.md)的语义背景色。

## 交互与状态

可点击的列表项必须可通过键盘到达并显示焦点。行内操作按钮（编辑、删除）遵循[无障碍](../foundations/accessibility.md)的触摸目标与焦点规则。

无数据、加载、失败状态必须呈现明确状态及后续动作（引导创建、重试）。状态过渡和减少动效遵循[动效与性能](../foundations/motion-performance.md)。

窄视口按[响应式与移动端](../foundations/responsive-mobile.md)收紧内边距，保持触摸目标合规。行内操作按钮在窄视口可折叠或垂直排列。

## 与表格的协作

列表卡片与表格可以出现在同一产品中：
- 客户详情 Sheet 内的实体列表使用列表卡片
- 客户列表页、商机列表页使用表格
- 从列表卡片点击项可导航到完整列表页（表格视图）

两者共享相同的设计令牌（颜色、字号、间距）以确保视觉一致性。
