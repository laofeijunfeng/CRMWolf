# 表格

- **用途：**展示可比较、可扫描的信息密集型记录。
- **适用范围：**components 领域。
- **权威性：**本主题是表格表头、行、对齐与交互的唯一事实来源。
- **相关规范：**[组件索引](README.md) · [列表卡片](list-card.md)（受限空间替代方案） · [间距与布局](../foundations/spacing-layout.md) · [排版](../foundations/typography.md) · [颜色令牌](../foundations/color-tokens.md) · [动效与性能](../foundations/motion-performance.md) · [无障碍](../foundations/accessibility.md)

## 结构与内容

表头说明每列的含义；列标题简短、稳定，并与单元格内容一一对应。记录名称和主要识别信息置于便于扫描的位置，操作集中在操作列或行内菜单，避免每格都成为不同入口。

表格仅用于跨行比较有价值的数据。内容较长时允许行随内容扩展，不得以固定行高截断关键信息。无记录、加载、失败和筛选无结果必须呈现明确状态及后续动作。

## 表头、行与对齐

表头、数据行和单元格的高度与内边距使用[间距与布局](../foundations/spacing-layout.md)的表格令牌。表头以辅助信息层级区分于数据；行之间以轻量分隔组织，不使用竖向分隔线制造网格。

文本按阅读方向对齐；数值、金额、百分比与日期在同列保持一致对齐方式，并使比较重点容易扫读。行悬停仅用于可指向环境的可交互反馈，使用[颜色令牌](../foundations/color-tokens.md)的语义背景；不应让悬停成为发现操作的唯一方式。

## 操作与响应式

排序、选择、展开和行级操作必须有可理解的名称、状态和键盘路径。遵循[无障碍](../foundations/accessibility.md)的表格语义、焦点和动态反馈规则。

窄视口优先保留任务所需列；超出空间时仅表格容器可横向滚动，页面主体不得横向滚动。遵循[响应式与移动端](../foundations/responsive-mobile.md)。悬停与状态变化遵循[动效与性能](../foundations/motion-performance.md)。
