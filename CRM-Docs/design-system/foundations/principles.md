# 设计原则

- **用途：**目标架构与设计决策优先级。
- **适用范围：**foundations 领域。
- **权威性：**本主题是该范围的唯一事实来源。
- **相关规范：**[foundations 索引](README.md) · [设计系统根入口](../README.md)

## 目标架构

本组基础规范定义目标架构，不声明其已经由任何现有界面实现。后续文档以本领域主题为跨页面、跨组件规则的唯一来源。

## 核心价值

| 原则 | 决策含义 |
| --- | --- |
| 极致克制 | 每个视觉与交互元素应有明确用途，避免装饰性干扰。 |
| 柔和低噪 | 以中性、低饱和层次承载信息，不以强烈视觉夺取注意。 |
| 统一有序 | 使用已定义的语义令牌，保持相同语义的表达一致。 |
| 服务内容 | 内容与任务优先于界面装饰和品牌表现。 |

## 决策优先级

出现取舍时，依次保障：功能完整性、可用性、性能、视觉一致性、装饰性。前一项受到影响时，不以后一项交换。

## 使用边界

- 跨主题规则应链接其唯一所有者，而不是在页面或组件文档重复。
- 令牌的颜色见[颜色令牌](color-tokens.md)，几何见[间距与布局](spacing-layout.md)。
