# 写作规范

- **用途：**约束设计系统主题文档的结构与边界。
- **适用范围：**新增或修改的 Markdown 主题文件。
- **权威性：**本文件拥有文档写作规则。
- **相关规范：**[文档优先级](document-precedence.md) · [质量检查](quality-checks.md)

## 必需元数据头

每份主题文档开头必须说明：用途、适用范围、权威性或优先级、相关规范。内容保持目标状态，并在未落地处明确说明。

## 一个主题，一个所有者

每份文件只拥有一个主题。规则由唯一事实来源定义；需要上下游信息时链接到所有者，而非复制内容。

## 相对链接

目录内导航必须使用相对 Markdown 链接。新增主题后，领域索引必须链接它，并保持从根入口可达。

## 禁止重复

不得在索引、模式或组件文件重述其他主题拥有的令牌、状态、数值或实现规则。修订规则时更新其所有者，并检查引用链接。
