Part 5: 设计要点汇总

Human-in-the-Loop 核心设计
  • Signature Element: 确认节点内嵌设计，不占用独立空间
  • 风险指示器: 左侧竖线颜色/宽度表示风险等级
  • 状态演变: 等待 → 已确认 → 执行中 → 完成（可见）
  • 参数紧凑: 使用 "key: value" 格式，不占用大空间
  • 候选卡片: 单选按钮 + 实体信息 + hover/focus 反馈
  • 补全表单: 输入框 + 搜索按钮，嵌入在节点中
无障碍规范
  • ✓ 所有可交互元素 cursor: pointer
  • ✓ Hover 无布局偏移
  • ✓ Focus 状态可见（outline 2px）
  • ✓ ARIA 属性完整（radiogroup, radio, alertdialog）
  • ✓ 键盘导航完整（↑↓←→ Enter Escape）
  • ✓ 尊重 prefers-reduced-motion
Empty/Loading/Error 状态
  • ✓ 搜索加载中：旋转动画 + aria-live
  • ✓ 搜索无结果：温暖提示 + CTA 按钮
  • ✓ 搜索失败：错误提示 + 重试按钮
  • ✓ 实体不存在：建议其他关键词
信任设计原则
  • ✓ 进度可见性：用户永远知道 "AI 在做什么"
  • ✓ 结果透明度：ThinkingBubble 显示思考过程
  • ✓ 可控感：用户可随时收起/展开/取消
  • ✓ 风险可见性：左侧竖线表示风险等级
  • ✓ 状态演变可见：等待 → 确认 → 执行 → 完成
风险等级指示器规范
风险等级 左侧竖线颜色 竖线宽度 说明 示例操作
低风险 #2B633C 2px（细线） 暗示安全 创建跟进记录、查看客户
