## 一、合规问题报告模板

当 AI 遇到无法合规的情况，必须使用此模板报告：

```markdown
## 合规问题报告

### 问题类型（必填）
- [ ] TypeScript 违规（any/as/@ts-ignore/!）
- [ ] 架构违规（错误文件位置、跨层调用）
- [ ] 测试缺失（无单测、无 Stories）
- [ ] 文档缺失（文档未同步更新）
- [ ] 权限违规（使用未定义的权限码）
- [ ] 配置锁定（需修改被锁定的配置）
- [ ] 其他：___

### 受影响文件（必填）
- 文件路径：___
- 行号：___

### 问题描述（必填）
[清晰描述违规内容]

### 尝试修复（必填）
[AI 尝试了什么方法]

### 阻塞原因（必填）
[为什么无法继续]

### 需人工处理（必填）
[需要人类做什么]

### 相关文档
- [ ] TYPESCRIPT.md
- [ ] ARCHITECTURE.md
- [ ] TESTING.md
- [ ] GLOSSARY.md
- [ ] 其他：___

### 报告时间
___
```

## 二、合规检查清单

### 2.1 Pre-Commit 检查

| 检查项 | 工具 | 失败处理 |
|--------|------|----------|
| TypeScript 四禁令 | ESLint custom rules | 阻止提交 |
| TypeScript 类型完整性 | vue-tsc --noEmit | 阻止提交 |
| 配置文件锁定 | Git hooks | 阻止提交 |
| Props 类型完整性 | ESLint Vue rules | 阻止提交 |

### 2.2 Pre-Push 检查

| 检查项 | 工具 | 失败处理 |
|--------|------|----------|
| 单元测试通过 | Vitest/pytest | 阻止推送 |
| 覆盖率达标 | coverage | 阻止推送 |
| Stories 文件存在 | 文件检测 | 阻止推送 |

### 2.3 CI 检查

| 检查项 | 工具 | 失败处理 |
|--------|------|----------|
| 全量 ESLint | ESLint | 阻止合并 |
| 全量类型检查 | vue-tsc/mypy | 阻止合并 |
| 全量测试 | Vitest/pytest | 阻止合并 |
| 文档同步检查 | doc-sync script | 阻止合并 |
| 合规基线检查 | compliance script | 阻止合并 |

## 三、违规处理流程

```
发现违规
    │
    ├── AI 可修复
    │       │
    │       └── 修复代码 → 重新校验 → 通过 → 继续
    │
    └── AI 无法修复
            │
            ├── 填写合规报告
            │       │
            │       └── 提交报告 → 等待人工审批
            │               │
            │               ├── 审批通过 → 执行 → 继续
            │               └── 审批拒绝 → 调整方案 → 重新尝试
            │
            └── 停止执行（不尝试绕开红线）
```

## 四、红线违规处理

### 4.1 TypeScript 禁令违规

| 违规 | 自动处理 | 人工处理 |
|------|----------|----------|
| 新增 `: any` | ESLint 报错阻止提交 | 需审批 |
| 新增 `as any` | ESLint 报错阻止提交 | 需审批 |
| 新增 `@ts-ignore` | ESLint 报错阻止提交 | 需审批 |
| 新增 `!` 非空断言 | ESLint 报错阻止提交 | 需审批 |

### 4.2 配置修改违规

| 违规 | 自动处理 | 人工处理 |
|------|----------|----------|
| 修改 tsconfig.json | Git hooks 阻止提交 | 需审批 |
| 修改 eslint.config.js | Git hooks 阻止提交 | 需审批 |
| 修改 pyproject.toml | Git hooks 阻止提交 | 需审批 |

## 五、合规基线管理

### 5.1 存量违规跟踪

合规基线文件：`CRM-Client/.compliance-baseline.json`

```json
{
  "generatedDate": "2026-04-21",
  "typeScriptViolations": {
    ":any": { "total": 158, "files": { ... } },
    "as any": { "total": 119, "files": { ... } }
  },
  "policy": {
    "newCode": "ZERO_TOLERANCE",
    "existingCode": "TRACKED",
    "baselineReductionTarget": "60%"
  }
}
```

### 5.2 新代码零容忍

- 新文件：100% 合规，零违规
- 新增代码：零新增违规
- 违规计数：仅统计存量，不统计新增

## 六、合规报告提交方式

1. **创建报告文件**：`CRM-Docs/compliance-reports/[date]-[issue-type].md`
2. **填写模板**：使用上述模板填写完整
3. **提交 PR**：将报告作为单独 PR 提交
4. **等待审批**：人工审批后执行
