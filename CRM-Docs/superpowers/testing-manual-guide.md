# AI 实体解析架构 - 手动测试指南

## 测试环境准备

### 1. 启动后端
```bash
cd CRM-Server
./run.sh
```

### 2. 启动前端
```bash
cd CRM-Client
npm run dev
```

### 3. 打开浏览器
访问 `http://localhost`

## 测试用例 1：线索创建（验证原有功能）

**目的**：验证迁移后的线索 AI 创建功能未被破坏

**步骤**：
1. 打开线索管理页面 (`/leads`)
2. 点击 "AI 创建线索" 按钮
3. 输入测试数据：
   ```
   张三，13800138000，来自杭州的阿里巴巴，大概500人，网上注册来的，想做电商系统，下一步推进POC部署试用，下周三再联系
   ```
4. 点击 "智能识别"
5. **验证解析结果**：
   - ✅ 线索名称：阿里巴巴
   - ✅ 来源：线上注册
   - ✅ 城市：杭州
   - ✅ 规模：501-1000人
   - ✅ 联系人：张三
   - ✅ 电话：13800138000
   - ✅ 跟进内容：想做电商系统
   - ✅ 下一步动作：推进POC部署试用
6. 点击 "创建线索"
7. **验证创建成功**：
   - ✅ 跳转到线索详情页
   - ✅ 线索信息正确
   - ✅ 跟进记录已自动创建

**期望结果**：功能与原来完全一致，无破坏

## 测试用例 2：客户创建（验证新功能）

**目的**：验证新增的客户 AI 创建功能

**步骤**：
1. 打开客户管理页面 (`/customers`)
2. 点击 "AI 创建客户" 按钮（新增）
3. 输入测试数据：
   ```
   阿里巴巴，杭州，张三 13800138000 技术总监 zhangsan@alibaba.com，大概500人，网上注册来的，互联网公司，想做电商系统，下周三再联系
   ```
4. 点击 "智能识别"
5. **验证解析结果**：
   - ✅ 客户名称：阿里巴巴
   - ✅ 城市：杭州
   - ✅ 规模：501-1000人
   - ✅ 来源：线上注册
   - ✅ 行业提示：互联网
   - ✅ 联系人：张三
   - ✅ 电话：13800138000
   - ✅ 职务：技术总监
   - ✅ 邮箱：zhangsan@alibaba.com
6. 点击 "创建客户"
7. **验证创建成功**：
   - ✅ 跳转到客户详情页
   - ✅ 客户信息正确
   - ✅ 主联系人已创建
   - ✅ 档案状态：GENERATING（异步生成中）

9. **验证档案生成**：
