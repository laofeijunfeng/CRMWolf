# 客户管理模块 - 功能说明

> 版本：1.1 | 更新日期：2026-06-01

---

## 一、模块概述

客户管理模块是 CRMWolf 系统的核心模块之一，用于管理已确认购买意向的企业客户。客户可通过线索转化或手动创建两种方式产生，支持多联系人管理、客户档案 AI 生成、公海池管理等功能。

### 1.1 核心价值

| 特性 | 说明 |
|------|------|
| **360° 客户视图** | 客户信息、联系人、跟进记录、商机、合同、回款、发票一站式查看 |
| **AI 智能档案** | 自动生成企业背景、主营业务、公司官网等信息 |
| **公海池机制** | 支持客户退回公海、领取、分配，实现资源流转 |
| **多联系人管理** | 支持主联系人、决策人标记、组织架构关系 |
| **状态流转** | 跟进中 → 赢单/输单/失效，完整生命周期管理 |

### 1.2 模块定位

```
线索转化 → 客户创建 → 客户跟进 → 商机推进 → 合同签署
                ↓
           公海池流转（退回/领取/分配）
```

### 1.3 核心业务链路

```
线索模块 ──转化──→ 客户模块 ──创建商机──→ 商机模块
                      │                      │
                      ↓                      ↓
               联系人管理              合同签署
                      │                      │
                      └───────┬──────────────┘
                              ↓
                         回款/发票
```

## 二、核心功能

### 2.1 客户创建

| 方式 | 说明 | 特点 |
|------|------|------|
| **线索转化** | 从合格线索转化为客户 | 自动创建主联系人、AI 生成档案、分析线索跟进记录 |
| **手动创建** | 直接创建新客户 | 需填写客户名称、城市等必填信息，AI 自动生成档案 |

**线索转化流程**：
1. 选择合格线索
2. 补充客户名称（可选，默认使用线索公司名）
3. 补充地址信息（可选）
4. 设置默认采购方式（可选）
5. 调用转化接口
6. 系统自动：创建客户 + 创建主联系人 + 触发 AI 档案生成 + 发送飞书通知

### 2.2 客户档案（AI 生成）

客户档案由 AI 自动生成，包含以下信息：

| 字段 | 说明 | 数据来源 |
|------|------|----------|
| `company_background` | 企业背景 | AI 根据客户名称搜索生成 |
| `company_website` | 公司官网 | AI 搜索获取 |
| `main_business` | 主营业务 | AI 分析生成 |
| `similar_customers` | 同行业客户 | AI 匹配现有客户库 |
| `project_background` | 项目需求背景 | 从线索跟进记录分析提取 |
| `industry` | 所属行业 | AI 推断（可手动修改） |

**档案状态**：
| 状态 | 说明 |
|------|------|
| `PENDING` | 等待生成（刚创建） |
| `GENERATING` | 正在生成（预计 10-30 秒） |
| `COMPLETED` | 生成完成 |
| `FAILED` | 生成失败（可重新生成） |

**档案生成流程**：
```
客户创建 → PENDING → 触发异步任务 → GENERATING
    ↓
AI 判断行业（如：金融、互联网）
    ↓
AI 搜索企业信息（背景、官网、主营业务）
    ↓
AI 匹配同行业客户（从现有客户库）
    ↓
AI 分析线索跟进记录（项目需求背景）
    ↓
COMPLETED（生成完成）
```

**重新生成档案**：
- 用户可手动触发重新生成（POST /{customer_id}/regenerate-profile）
- 生成失败时自动记录错误原因
- 档案内容可手动编辑覆盖

### 2.3 联系人管理

| 功能 | 说明 |
|------|------|
| **添加联系人** | 姓名、手机号必填，支持职务、邮箱、微信等 |
| **主联系人设置** | 每个客户仅一个主联系人 |
| **决策人标记** | 标记关键决策人，影响销售策略 |
| **组织架构** | 通过 `reports_to` 字段建立汇报关系 |
| **联系人编辑/删除** | 支持修改联系人信息、删除联系人 |

**联系人字段**：
```python
name: str              # 姓名（必填）
gender: int            # 性别：0=未知, 1=男, 2=女
position: str          # 职务（如：CTO、采购经理）
is_decision_maker: bool # 是否关键决策人
mobile: str            # 手机号（必填）
email: str             # 邮箱
wechat_id: str         # 微信ID
remark: str            # 备注
reports_to: int        # 汇报对象联系人ID
is_primary: bool       # 是否主联系人
```

### 2.4 公海池管理

公海池用于存放无负责人的客户，实现资源流转。

| 功能 | 说明 | 权限要求 |
|------|------|----------|
| **退回公海** | 将客户退回公海池，解除负责人绑定 | 负责人本人或销售总监/管理员 |
| **领取客户** | 从公海池领取客户，成为负责人 | 所有销售人员 |
| **分配客户** | 管理员/总监将客户分配给指定人员 | TEAM_ADMIN 或 SALES_DIRECTOR |

**退回原因枚举**：
| 值 | 说明 |
|------|------|
| `丢单` | 客户已选择竞争对手 |
| `无意向` | 客户明确表示无合作意向 |
| `信息错误` | 客户信息不准确或无效 |
| `长期未跟进` | 超过规定时间未跟进 |
| `预算不足` | 客户预算无法匹配产品价格 |
| `其他` | 其他原因 |

### 2.5 客户状态管理

| 状态值 | 名称 | 说明 |
|--------|------|------|
| `0` | 跟进中 | 正常跟进状态 |
| `1` | 已赢单 | 成功成交，可创建合同 |
| `2` | 已输单 | 失去客户，需记录输单原因 |
| `3` | 已失效 | 公海池客户或失效客户 |

**状态操作**：
- **标记赢单**：状态变为 1，发送飞书通知
- **标记输单**：状态变为 2，必须填写输单原因，发送飞书通知
- **标记失效**：状态变为 3

**状态流转图**：
```
                    ┌─────────────┐
                    │   跟进中    │
                    │    (0)      │
                    └──────┬──────┘
                           │
           ┌───────────────┼───────────────┐
           │               │               │
           ↓               ↓               ↓
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │   已赢单    │ │   已输单    │ │   已失效    │
    │    (1)      │ │    (2)      │ │    (3)      │
    └──────┬──────┘ └─────────────┘ └──────┬──────┘
           │                              │
           ↓                              ↓
    创建合同/商机                     公海池领取
           │                              │
           └──────────────────────────────┘
                           │
                           ↓
                    ┌─────────────┐
                    │   跟进中    │
                    │    (0)      │
                    └─────────────┘
```

### 2.6 关联数据查询

客户详情页支持查看以下关联数据：

| Tab | 内容 |
|------|------|
| **跟进** | 跟进记录时间线，支持添加、编辑、删除 |
| **联系人** | 客户所有联系人 |
| **商机** | 客户关联的所有商机 |
| **合同** | 客户的所有合同 |
| **回款** | 客户的回款计划和回款记录 |
| **发票** | 客户的发票申请和发票抬头 |

## 六、业务规则

### 6.1 客户名称唯一性

- 客户名称（`account_name`）全局唯一
- 创建/编辑时校验名称是否存在
- 归一化名称（`account_name_norm`）用于模糊匹配（去除后缀如"有限公司"、"股份有限公司"等）

### 6.2 负责人规则

- 创建客户时，未指定负责人则默认为创建人
- 退回公海时，解除负责人绑定（`owner_id` 设为 null）
- 领取/分配时，设置新的负责人

### 6.3 删除规则

- 逻辑删除，不物理删除
- 需检查权限：
  - 管理员可删除所有客户
  - 销售人员只能删除自己负责的客户

### 6.4 档案生成规则

- 客户创建后自动触发档案生成
- 档案生成异步执行，预计 10-30 秒
- 生成失败可重新触发
- 档案内容可手动编辑

### 6.5 公海流转规则

- 退回公海必须填写退回原因和详细说明
- 退回后发送飞书通知
- 领取客户后成为负责人
- 管理员/总监可强制分配客户

### 6.6 团队数据隔离

- 所有查询自动添加 `team_id` 过滤条件
- 跨团队数据不可见（包括公海池）
- API层自动注入当前用户的团队ID
- 联系人同样遵循团队隔离规则

### 6.7 转化关联规则

- 线索转化后记录 `source_lead_id` 关联原线索
- 删除客户时同步恢复线索状态为 NEW
- 客户的跟进记录继承线索的跟进历史

## 八、与其他模块的关联

| 关联模块 | 关系 | 说明 |
|----------|------|------|
| **线索管理** | 来源关系 | `source_lead_id` 记录来源线索，删除客户时恢复线索状态 |
| **联系人管理** | 子表关系 | 一个客户可有多个联系人，一个主联系人 |
| **商机管理** | 业务关系 | 一个客户可有多个商机 |
| **合同管理** | 业务关系 | 一个客户可有多个合同，删除客户前需先删除合同 |
| **回款管理** | 财务关系 | 通过合同关联回款计划和回款记录 |
| **发票管理** | 财务关系 | 发票申请、发票抬头 |
| **采购方式** | 配置关系 | `default_procurement_method_id` 关联采购方式 |
| **行业管理** | 分类关系 | `industry` 关联行业分级体系 |
| **操作日志** | 记录关系 | 记录客户创建、转化、输单、退回公海等操作 |
| **飞书通知** | 通知关系 | 转化、领取、分配、赢单、输单、退回公海时发送通知 |
