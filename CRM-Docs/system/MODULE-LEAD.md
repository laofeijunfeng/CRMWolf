# 线索管理模块 - 功能说明

> 版本：1.0 | 更新日期：2026-06-01

---

## 一、模块概述

线索管理模块是 CRMWolf 系统的销售漏斗入口，用于管理潜在客户线索。线索可通过手动创建、AI智能创建、批量导入三种方式产生，支持跟进记录、公海池流转、线索转化等完整生命周期管理。

### 1.1 核心价值

| 特性 | 说明 |
|------|------|
| **多渠道线索录入** | 手动创建、AI智能解析、批量导入（最多100条） |
| **跟进记录管理** | 记录跟进方式、内容、下一步动作，支持时间线展示 |
| **公海池机制** | 支持线索退回公海、领取、分配，实现资源流转 |
| **线索转化** | 合格线索一键转化为客户，自动创建主联系人、迁移跟进记录 |
| **AI智能创建** | 自然语言解析线索信息，智能填充表单 |
| **待跟进提醒** | 自动识别超期未跟进线索，支持飞书通知 |

### 1.2 模块定位

```
线索录入 → 分配负责人 → 跟进记录 → 有效性判断 → 转化/无效
    ↓           ↓            ↓               ↓
  公海池 ← 退回公海 ← 长期未跟进 ←───────┘
                                    ↓
                            客户管理模块
```

## 二、核心功能

### 2.1 线索创建

| 方式 | 说明 | 特点 |
|------|------|------|
| **手动创建** | 填写表单创建线索 | 必填：线索名称、来源、城市、联系人、手机号 |
| **AI智能创建** | 自然语言解析创建 | 输入一段描述，AI自动提取线索信息 |
| **批量导入** | 批量导入线索 | 最多100条，自动校验手机号唯一性 |

**AI智能创建流程**：
1. 用户输入自然语言描述（如："张三，北京，来自市场活动，电话13800138000"）
2. AI 解析提取字段：线索名称、联系人、城市、来源、手机号
3. 用户确认或修改
4. 创建线索

**创建规则**：
- 创建人自动成为负责人（`owner_id = creator_id`）
- 手机号全局唯一（跨团队唯一性校验）
- 状态默认为"新建"（NEW = 0）

### 2.2 线索跟进

| 功能 | 说明 |
|------|------|
| **添加跟进记录** | 记录跟进方式、内容、下一步动作 |
| **跟进方式** | 电话、微信、拜访、邮件、其他 |
| **下次跟进** | 可设置计划下次跟进时间和动作 |
| **时间线展示** | 详情页按时间倒序展示跟进历史 |

**跟进记录字段**：
```python
content: str           # 跟进内容（必填）
method: FollowUpMethod # 跟进方式：电话/微信/拜访/邮件/其他
next_follow_time: datetime  # 计划下次跟进时间
next_action: str       # 下一步动作内容
```

### 2.3 公海池管理

公海池用于存放无负责人的线索，实现资源流转。

| 功能 | 说明 | 权限要求 |
|------|------|----------|
| **退回公海** | 将线索退回公海池，解除负责人绑定 | 负责人本人或销售总监/管理员 |
| **领取线索** | 从公海池领取线索，成为负责人 | 所有销售人员 |
| **分配线索** | 管理员/总监将线索分配给指定人员 | TEAM_ADMIN 或 SALES_DIRECTOR |

**公海规则**：
- `owner_id = null` 为公海线索
- 领取后 `owner_id` 设置为当前用户
- 分配后 `owner_id` 设置为指定用户

### 2.4 线索转化

将合格线索转化为客户，进入正式销售流程。

**转化流程**：
1. 校验线索状态（仅 NEW/FOLLOWING 可转化）
2. 补充客户信息：
   - 客户名称（必填）
   - 联系人姓名（默认使用线索联系人）
   - 联系人手机（默认使用线索手机）
   - 地址（可选）
   - 行业（可选）
3. 系统自动执行：
   - 创建客户记录
   - 创建主联系人
   - 迁移线索跟进记录到客户
   - 触发 AI 生成客户档案
   - 发送飞书通知
4. 更新线索状态为 CONVERTED

**转化限制**：
- 已转化的线索不能再次转化
- 已转化的线索删除需先删除关联客户

### 2.5 线索状态管理

| 状态值 | 名称 | 说明 |
|--------|------|------|
| `0` | 新建 | 刚创建，未开始跟进 |
| `1` | 跟进中 | 正在跟进中 |
| `2` | 已转化 | 已转化为客户 |
| `3` | 无效 | 标记为无效线索 |

**状态操作**：
- **开始跟进**：状态从 0 变为 1
- **标记无效**：状态变为 3，必须填写无效原因
- **转化成功**：状态变为 2

**状态流转图**：
```
新建(0) ──添加跟进──→ 跟进中(1) ──转化──→ 已转化(2)
   │                    │
   └──标记无效──────────┴────────→ 无效(3)
```

### 2.6 待跟进提醒

系统自动识别需要跟进的线索：

| 条件 | 说明 |
|------|------|
| **待跟进** | 有计划的下次跟进时间，且时间已过 |
| **长期未跟进** | 超过规定时间（如7天）未添加跟进记录 |

**API接口**：
- `GET /api/v1/leads/follow-up/reminder` - 获取待跟进线索列表

## 三、数据模型

### 3.1 线索表（crm_leads）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BigInteger | 主键 |
| `team_id` | BigInteger | 团队ID（数据隔离） |
| `lead_name` | String(255) | 线索名称（必填） |
| `source` | Enum | 线索来源 |
| `city` | String(100) | 所在城市（必填） |
| `contact_name` | String(100) | 联系人姓名（必填） |
| `contact_phone` | String(20) | 联系人手机（必填，唯一） |
| `company_scale` | Enum | 公司规模 |
| `owner_id` | String(100) | 负责人ID（飞书用户ID，null为公海） |
| `status` | Enum | 线索状态：0/1/2/3 |
| `invalid_reason` | String(500) | 无效原因 |
| `pool_id` | BigInteger | 所属线索池ID |
| `creator_id` | String(100) | 创建人ID |
| `created_time` | DateTime | 创建时间 |
| `last_modified_time` | DateTime | 最后修改时间 |
| `version` | Integer | 版本号（乐观锁） |

**索引**：
- `idx_team_id` - 团队ID索引（数据隔离）
- `idx_owner_id` - 负责人索引（公海/我的筛选）
- `idx_status` - 状态索引
- `idx_source` - 来源索引
- `idx_city` - 城市索引
- `idx_created_time` - 创建时间索引

### 3.2 线索跟进记录表（crm_lead_follow_ups）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BigInteger | 主键 |
| `team_id` | BigInteger | 团队ID |
| `lead_id` | BigInteger | 关联线索ID（外键） |
| `content` | Text | 跟进内容（必填） |
| `method` | Enum | 跟进方式 |
| `next_follow_time` | DateTime | 计划下次跟进时间 |
| `next_action` | Text | 下一步动作内容 |
| `creator_id` | String(100) | 创建人ID |
| `created_time` | DateTime | 创建时间 |

### 3.3 线索来源枚举（LeadSource）

| 值 | 说明 |
|------|------|
| `线上注册` | 客户在线注册 |
| `市场活动` | 市场营销活动获取 |
| `客户推荐` | 现有客户推荐 |
| `电话营销` | 电话营销获取 |
| `网站咨询` | 网站咨询获取 |
| `展会` | 展会获取 |
| `其他` | 其他来源 |

### 3.4 公司规模枚举（CompanyScale）

| 值 | 说明 |
|------|------|
| `1-50人` | 小微企业 |
| `51-200人` | 中型企业 |
| `201-500人` | 中大型企业 |
| `501-1000人` | 大型企业 |
| `1000人以上` | 超大型企业 |

### 3.5 跟进方式枚举（FollowUpMethod）

| 值 | 说明 |
|------|------|
| `电话` | 电话沟通 |
| `微信` | 微信沟通 |
| `拜访` | 上门拜访 |
| `邮件` | 邮件沟通 |
| `其他` | 其他方式 |

## 五、前端页面

### 5.1 线索列表页（Leads.vue）

**路径**：`/leads`

**功能**：
- 快捷筛选 Tab：全部线索、我的线索、公海线索、待跟进
- 表格筛选：线索名称、来源、城市、状态
- 动态排序：创建时间、线索名称、城市、状态

| 删除线索 | `lead:delete:all` 或 `lead:delete:own`（自己负责） |
| 转化线索 | `lead:convert` |
| 领取线索 | `lead:claim` |
| 分配线索 | `lead:assign` |
| 退回公海 | `lead:return` |

### 5.2 线索详情页（LeadDetail.vue）

**路径**：`/leads/:id`

**功能**：
- 基本信息卡片：线索名称、状态、来源、城市、公司规模、联系人、负责人
- 跟进记录时间线：按时间倒序展示跟进历史
- 操作按钮：
  - 编辑、添加跟进
  - 转化、标记无效
  - 分配、退回公海、删除

### 5.3 线索转化页（LeadConvert.vue）

**路径**：`/leads/:id/convert`

**功能**：
- 预填线索信息（联系人、手机号）
- 补充客户信息表单：
  - 客户名称（必填）
  - 地址（可选）
  - 行业（可选）
  - 采购方式（可选）
- 转化确认

### 5.4 AI智能创建对话框（AILeadCreateDialog.vue）

**功能**：
- 输入自然语言描述
- AI 解析并填充表单

## 六、业务规则

### 6.1 手机号唯一性

- 联系人手机号全局唯一（跨团队）
- 创建/导入时校验手机号是否已存在
- 已存在则返回错误提示

### 6.2 负责人规则

- 创建线索时，创建人自动成为负责人
- 退回公海时，解除负责人绑定（`owner_id` 设为 null）
- 领取/分配时，设置新的负责人

### 6.3 状态流转规则

- 新建线索可添加跟进，状态变为"跟进中"
- 跟进中线索可转化或标记无效
- 已转化线索不能修改状态
- 无效线索可重新激活（添加跟进）

### 6.4 删除规则

- 删除线索时同步删除关联跟进记录
- 已转化的线索删除前需先删除关联客户
- 删除后恢复线索状态为 NEW

### 6.5 团队数据隔离

- 所有查询自动添加 `team_id` 过滤条件
- 跨团队数据不可见
- 公海池按团队隔离

## 七、权限码

| 权限码 | 说明 |
|------|------|
| `lead:create` | 创建线索 |
| `lead:view:own` | 查看自己负责的线索 |
| `lead:view:all` | 查看所有线索 |
| `lead:edit:own` | 编辑自己负责的线索 |
| `lead:edit:all` | 编辑所有线索 |
| `lead:delete:own` | 删除自己负责的线索 |
| `lead:delete:all` | 删除所有线索 |
| `lead:convert` | 转化线索 |
| `lead:claim` | 领取线索 |
| `lead:assign` | 分配线索 |
| `lead:return` | 退回线索到公海 |

**角色权限**：
| 角色 | 权限范围 |
|------|----------|
| TEAM_ADMIN | 查看所有、编辑所有、删除所有、分配 |
| SALES_DIRECTOR | 查看所有、编辑所有、删除所有、分配 |
| SALES_MEMBER | 仅查看、编辑、删除自己负责的线索 |
