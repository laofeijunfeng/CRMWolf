## 一、系统概述

### 1.1 系统简介

CRMWolf 是一套面向企业的客户关系管理（CRM）系统，旨在帮助企业高效管理销售全流程，从线索获取、客户跟进、商机推进到合同签订、回款管理、发票开具，形成完整的销售闭环。

### 1.2 核心价值

| 特性 | 说明 |
|------|------|
| **全流程管理** | 覆盖线索 → 客户 → 商机 → 合同 → 回款 → 发票完整链路 |
| **精细化权限** | 基于 RBAC 的权限体系，支持数据范围控制 |
| **审批流程** | 合同审批、发票审批，支持多级审批 |
| **数据隔离** | 销售人员仅查看自己负责的数据，管理层可查看全部 |
| **操作审计** | 完整记录所有操作日志，支持追溯 |

### 1.3 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| **前端** | Vue 3 + TypeScript + Element Plus + Pinia + Vite | Vue 3.5, TS 5.7 |
| **后端** | Python + FastAPI + SQLAlchemy + Pydantic | Python 3.11, FastAPI 0.115 |
| **数据库** | MySQL + Redis | MySQL 8.0, Redis 7.x |
| **认证** | 邮箱验证码 + 密码 + JWT | 阿里云邮件推送 |
| **邮件服务** | 阿里云 DirectMail API | 免费额度 200封/天 |

## 四、业务流程

### 4.1 销售全流程

```
线索 → 客户 → 商机 → 合同 → 回款 → 发票
 ↓       ↓       ↓       ↓       ↓       ↓
转化    跟进    推进    审批    登记    申请
```

### 4.2 流程阶段说明

| 阶段 | 输入 | 输出 | 关键操作 |
|------|------|------|----------|
| 线索 | 市场活动、客户咨询 | 合格线索 | 录入、跟进、判定、转化 |
| 客户 | 线索转化 | 活跃客户 | 档案维护、跟进记录 |
| 商机 | 客户需求 | 赢单商机 | 阶段推进、报价、谈判 |
| 合同 | 赢单商机 | 执行合同 | 创建、审批、签订 |
| 回款 | 执行合同 | 回款记录 | 计划、登记、确认 |
| 发票 | 回款记录 | 开票记录 | 申请、审批、开票 |

### 4.3 线索转化流程

```
新线索 ──▶ 跟进联系 ──▶ 线索判定
                              │
              ┌───────────────┼───────────────┐
              │               │               │
              ▼               ▼               ▼
         无效线索         继续跟进         合格转化
         (关闭)          (跟进中)     (创建客户+商机)
```

### 4.4 商机推进流程

```
初步接洽 ──▶ 需求确认 ──▶ 方案报价 ──▶ 商务谈判 ──▶ 签订合同 ──▶ 赢单
                                                            │
                              ┌───────────────────────────────┘
                              │
                              ▼
                           输单
```

## 六、状态枚举

### 6.1 线索状态

| 值 | 名称 | 说明 |
|----|------|------|
| `NEW` | 新线索 | 刚录入 |
| `FOLLOWING` | 跟进中 | 正在跟进 |
| `CONVERTED` | 已转化 | 已转为客户 |
| `INVALID` | 无效 | 无效线索 |

### 6.2 客户状态

| 值 | 名称 | 说明 |
|----|------|------|
| `0` | 跟进中 | 正常跟进状态 |
| `1` | 已成交 | 已签约客户 |
| `2` | 已流失 | 失去客户 |
| `3` | 非激活 | 公海池客户 |

### 6.3 商机状态

| 值 | 名称 | 说明 |
|----|------|------|
| `0` | 跟进中 | 正在推进 |
| `1` | 已赢单 | 成功成交 |
| `2` | 已输单 | 失去商机 |

### 6.4 商机阶段

| 阶段 | 赢率 | 说明 |
|------|------|------|
| 初步接洽 | 10% | 初次接触客户 |
| 需求确认 | 20% | 明确客户需求 |
| 方案报价 | 40% | 提供解决方案 |
| 商务谈判 | 60% | 价格和条款谈判 |
| 签订合同 | 90% | 准备签约 |
| 赢单 | 100% | 成交 |

### 6.5 合同状态

| 值 | 说明 |
|------|------|
| `DRAFT` | 草稿 |
| `PENDING_REVIEW` | 待审批 |
| `SIGNED` | 已签订 |
| `EFFECTIVE` | 生效中 |
| `EXPIRED` | 已到期 |
| `TERMINATED` | 已终止 |

### 6.6 发票申请状态

| 值 | 说明 |
|------|------|
| `DRAFT` | 草稿 |
| `PENDING_REVIEW` | 待审批 |
| `APPROVED` | 已批准 |
| `REJECTED` | 已拒绝 |
| `ISSUED` | 已开票 |

## 七、API 接口规范

### 7.1 基础信息

| 项目 | 说明 |
|------|------|
| 基础路径 | `/api/v1` |
| 协议 | HTTP/HTTPS |
| 数据格式 | JSON |
| 字符编码 | UTF-8 |
| 认证方式 | JWT Bearer Token |

### 7.2 通用响应格式

**成功响应：**
```json
{
  "id": 1,
  "name": "数据名称",
  "created_time": "2024-01-01T00:00:00"
}
```

**错误响应：**
```json
{
  "detail": "错误信息描述"
}
```

**列表响应：**
```json
{
  "items": [...],
  "total": 100
}
```

### 7.3 核心接口列表

| 模块 | 接口路径 | 方法 | 说明 |
|------|----------|------|------|
| 认证 | `/auth/send-code` | POST | 发送邮箱验证码（注册/登录/重置密码） |
| 认证 | `/auth/register` | POST | 邮箱注册（验证码 + 姓名 + 密码可选） |
| 认证 | `/auth/login` | POST | 验证码登录 |
| 认证 | `/auth/login-password` | POST | 密码登录 |

| 线索 | `/api/v1/leads/` | GET/POST | 线索列表/创建 |

| 客户 | `/api/v1/customers/` | GET/POST | 客户列表/创建 |
| 客户 | `/api/v1/customers/{id}/contacts` | POST | 添加联系人 |
| 商机 | `/api/v1/opportunities/` | GET/POST | 商机列表/创建 |
| 商机 | `/api/v1/opportunities/{id}/advance` | POST | 推进阶段 |
| 商机 | `/api/v1/opportunities/{id}/win` | POST | 赢单 |
| 合同 | `/api/v1/contracts/` | GET/POST | 合同列表/创建 |
| 合同 | `/api/v1/contracts/{id}/review` | POST | 审批合同 |
| 回款 | `/api/v1/payment-plans/` | GET | 回款计划列表 |
| 回款 | `/api/v1/payment-records/{id}/confirm` | POST | 确认回款 |
| 发票 | `/api/v1/invoice-applications/` | GET/POST | 发票申请 |
| 发票 | `/api/v1/invoice-applications/{id}/review` | POST | 审批发票 |

## 八、系统架构

### 8.1 前端目录结构

```
CRM-Client/src/
├── api/                    # API 调用层
├── schemas/                # Zod 校验 Schema
├── components/             # 共享组件
├── composables/            # Composition API Hooks
├── directives/             # Vue 指令
├── router/                 # 路由配置
├── stores/                 # Pinia 状态管理
├── styles/                 # 样式文件
├── utils/                  # 工具函数
├── views/                  # 页面视图
└── main.ts                 # 入口文件
```

### 8.2 后端目录结构

```
CRM-Server/app/
├── api/                    # 路由层
├── core/                   # 核心配置
├── crud/                   # 数据访问层
├── models/                 # 数据库模型
├── schemas/                # Pydantic Schema
├── services/               # 业务逻辑层
├── constants/              # 常量定义
└── main.py                 # 应用入口
```

### 8.3 分层架构

**前端分层：**
| 层级 | 职责 |
|------|------|
| `api/` | HTTP 调用 + 类型导出 |
| `schemas/` | Zod 校验定义 |
| `components/` | UI 渲染 + 交互 |
| `stores/` | 全局状态管理 |
| `views/` | 页面编排 + 数据绑定 |

**后端分层：**
| 层级 | 职责 |
|------|------|
| `api/` | 请求解析 + 响应封装 |
| `schemas/` | 数据校验 + 序列化 |
| `crud/` | 数据库 CRUD |
| `services/` | 复杂业务逻辑 |
| `models/` | 表结构定义 |
