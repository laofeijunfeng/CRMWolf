# 线索管理模块 - 功能说明

> 版本：2.0 | 更新日期：2026-06-12

---

## 一、模块概述

线索管理模块是 CRMWolf 系统的销售漏斗入口，用于管理潜在客户线索。线索可通过手动创建、AI智能创建、批量导入三种方式产生，支持跟进记录、公海池流转、线索转化等完整生命周期管理。

### 1.1 核心价值

| 特性 | 说明 |
|------|------|
| **多渠道线索录入** | 手动创建、AI智能解析、批量导入（最多100条） |
| **跟进记录管理** | 记录跟进方式、内容、下一步动作，支持时间线展示 |
| **公海池机制** | 支持线索退回公海、领取、分配，实现资源流转 |
| **线索转化** | 合格线索一键转化为客户，自动创建主联系人、迁移跟进记录 |
| **AI智能创建** | 自然语言解析线索信息，智能填充表单 |
| **待跟进提醒** | 自动识别超期未跟进线索，支持飞书通知 |

### 1.2 模块定位

```
线索录入 → 分配负责人 → 跟进记录 → 有效性判断 → 转化/无效
    ↓           ↓            ↓               ↓
  公海池 ← 退回公海 ← 长期未跟进 ←───────┘
                                    ↓
                            客户管理模块
```

## 二、业务生命周期

### 2.1 线索生命周期图

```
新增线索 → 跟进中 → 已转化 → 客户
              ↓
           已输单 → 公海池（可重新领取）
```

### 2.2 线索状态说明

| 状态值 | 名称 | 说明 |
|--------|------|------|
| `0` / `NEW` | 新建 | 刚创建，未开始跟进 |
| `1` / `CONTACTING` | 跟进中 | 正在跟进中 |
| `2` / `CONVERTED` | 已转化 | 已转化为客户 |
| `3` / `LOST` / `INVALID` | 无效/输单 | 标记为无效或输单，退回公海 |

### 2.3 状态流转图

```
新建(0) ──添加跟进──→ 跟进中(1) ──转化──→ 已转化(2)
   │                    │
   └──标记无效──────────┴────────→ 无效(3)
```

### 2.4 状态操作

| 操作 | 状态变化 | 说明 |
|------|----------|------|
| 开始跟进 | 0 → 1 | 添加首次跟进记录 |
| 转化成功 | 1 → 2 | 线索转化为客户 |
| 标记无效 | 0/1 → 3 | 填写无效原因，退回公海 |
| 重新领取 | 3 → 1 | 从公海领取，重新跟进 |

## 三、核心功能

### 3.1 线索创建

| 方式 | 说明 | 特点 |
|------|------|------|
| **手动创建** | 填写表单创建线索 | 必填：线索名称、来源、城市、联系人、手机号 |
| **AI智能创建** | 自然语言解析创建 | 输入一段描述，AI自动提取线索信息 |
| **批量导入** | 批量导入线索 | 最多100条，自动校验手机号唯一性 |

**AI智能创建流程**：
1. 用户输入自然语言描述（如："张三，北京，来自市场活动，电话13800138000"）
2. AI 解析提取字段：线索名称、联系人、城市、来源、手机号
3. 用户确认或修改
4. 创建线索

**创建规则**：
- 创建人自动成为负责人（`owner_id = creator_id`）
- 手机号全局唯一（跨团队唯一性校验）
- 状态默认为"新建"（NEW = 0）
- 新增线索24小时内必须首次跟进

### 3.2 线索分配

**分配规则**：
1. **自动分配**：系统根据规则自动分配给销售人员
   - 按地区平均分配
   - 按销售人员的负载均衡
   
2. **手动分配**：销售总监手动分配线索给指定销售人员

**分配限制**：
- 每个销售人员同时最多拥有50个活跃线索
- 公海池中的线索可以被任何销售人员领取
- 销售人员每天最多领取10个公海线索

### 3.3 线索跟进

| 功能 | 说明 |
|------|------|
| **添加跟进记录** | 记录跟进方式、内容、下一步动作 |
| **跟进方式** | 电话、微信、拜访、邮件、其他 |
| **下次跟进** | 可设置计划下次跟进时间和动作 |
| **时间线展示** | 详情页按时间倒序展示跟进历史 |

**跟进记录字段**：
```python
content: str           # 跟进内容（必填）
method: FollowUpMethod # 跟进方式：电话/微信/拜访/邮件/其他
next_follow_time: datetime  # 计划下次跟进时间
next_action: str       # 下一步动作内容
```

**跟进提醒**：
- 超过3天未跟进的线索自动提醒
- 计划跟进时间到期前提醒

### 3.4 公海池管理

| 功能 | 说明 | 权限要求 |
|------|------|----------|
| **退回公海** | 将线索退回公海池，解除负责人绑定 | 负责人本人或销售总监/管理员 |
| **领取线索** | 从公海池领取线索，成为负责人 | 所有销售人员 |
| **分配线索** | 管理员/总监将线索分配给指定人员 | TEAM_ADMIN 或 SALES_DIRECTOR |

**公海规则**：
- `owner_id = null` 为公海线索
- 领取后 `owner_id` 设置为当前用户
- 分配后 `owner_id` 设置为指定用户
- 线索超过30天未跟进自动退回公海

### 3.5 线索转化

将合格线索转化为客户，进入正式销售流程。

**转化条件**：
- 客户有明确的采购意向
- 已经与决策人建立联系
- 了解客户需求和预算

**转化流程**：
1. 校验线索状态（仅 NEW/FOLLOWING 可转化）
2. 补充客户信息：
   - 客户名称（必填）
   - 联系人姓名（默认使用线索联系人）
   - 联系人手机（默认使用线索手机）
   - 地址（可选）
   - 行业（可选）
3. 系统自动执行：
   - 创建客户记录
   - 创建主联系人
   - 迁移线索跟进记录到客户
   - 触发 AI 生成客户档案
   - 发送飞书通知
4. 更新线索状态为 CONVERTED

**转化限制**：
- 已转化的线索不能再次转化
- 已转化的线索删除需先删除关联客户
- 线索转化周期建议不超过30天

### 3.6 线索输单

**输单原因**：
- 客户预算不足
- 客户选择了竞品
- 客户暂时没有需求
- 无法联系到客户
- 其他原因

**输单处理**：
1. 标记线索为"已输单"
2. 记录输单原因
3. 线索退回公海池
4. 其他销售人员可以重新领取

### 3.7 待跟进提醒

| 条件 | 说明 |
|------|------|
| **待跟进** | 有计划的下次跟进时间，且时间已过 |
| **长期未跟进** | 超过7天未添加跟进记录 |

**API接口**：
- `GET /api/v1/leads/follow-up/reminder` - 获取待跟进线索列表

## 四、数据模型

### 4.1 线索表（crm_leads）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BigInteger | 主键 |
| `team_id` | BigInteger | 团队ID（数据隔离） |
| `lead_name` | String(255) | 线索名称（必填） |
| `source` | Enum | 线索来源 |
| `city` | String(100) | 所在城市（必填） |
| `contact_name` | String(100) | 联系人姓名（必填） |
| `contact_phone` | String(20) | 联系人手机（必填，唯一） |
| `company_scale` | Enum | 公司规模 |
| `owner_id` | String(100) | 负责人ID（飞书用户ID，null为公海） |
| `status` | Enum | 线索状态：0/1/2/3 |
| `invalid_reason` | String(500) | 无效原因 |
| `pool_id` | BigInteger | 所属线索池ID |
| `creator_id` | String(100) | 创建人ID |
| `created_time` | DateTime | 创建时间 |
| `last_modified_time` | DateTime | 最后修改时间 |
| `version` | Integer | 版本号（乐观锁） |

**索引**：
- `idx_team_id` - 团队ID索引（数据隔离）
- `idx_owner_id` - 负责人索引（公海/我的筛选）
- `idx_status` - 状态索引
- `idx_source` - 来源索引
- `idx_city` - 城市索引
- `idx_created_time` - 创建时间索引

### 4.2 线索跟进记录表（crm_lead_follow_ups）

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BigInteger | 主键 |
| `team_id` | BigInteger | 团队ID |
| `lead_id` | BigInteger | 关联线索ID（外键） |
| `content` | Text | 跟进内容（必填） |
| `method` | Enum | 跟进方式 |
| `next_follow_time` | DateTime | 计划下次跟进时间 |
| `next_action` | Text | 下一步动作内容 |
| `creator_id` | String(100) | 创建人ID |
| `created_time` | DateTime | 创建时间 |

### 4.3 线索来源枚举（LeadSource）

| 值 | 说明 |
|------|------|

## 七、业务规则

### 7.1 手机号唯一性

- 联系人手机号全局唯一（跨团队）
- 创建/导入时校验手机号是否已存在
- 已存在则返回错误提示

### 7.2 负责人规则

- 创建线索时，创建人自动成为负责人
- 退回公海时，解除负责人绑定（`owner_id` 设为 null）
- 领取/分配时，设置新的负责人

### 7.3 分配与领取限制

- 销售人员最多拥有50个活跃线索
- 销售人员每天最多领取10个公海线索
- 线索超过30天未跟进自动退回公海

### 7.4 跟进要求

- 新增线索24小时内必须首次跟进
- 超过3天未跟进自动提醒

### 7.5 删除规则

- 删除线索时同步删除关联跟进记录
- 已转化的线索删除前需先删除关联客户
- 删除后恢复线索状态为 NEW

### 7.6 团队数据隔离

- 所有查询自动添加 `team_id` 过滤条件
- 跨团队数据不可见
- 公海池按团队隔离

## 十一、常见问题

**Q1: 线索转化后还能找回吗？**
A: 转化后的线索状态为"已转化"，不能恢复为跟进中状态，但可以查看转化记录。

**Q2: 如何重新分配已分配的线索？**
A: 目前只有销售总监有权限重新分配线索，需要先将线索退回公海池再重新分配。

**Q3: 公海池的线索有优先级吗？**
A: 系统默认按线索创建时间排序，最新创建的线索排在前面。

**Q4: 如何批量导入线索？**
A: 可以使用Excel模板批量导入线索，需要先下载模板，填写信息后上传。

**Q5: 线索输单后可以恢复吗？**
A: 输单后的线索退回公海池，可以重新领取，但历史记录会保留。
