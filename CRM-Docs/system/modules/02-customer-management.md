## 一、模块概述（技术视角）

客户管理模块是 CRMWolf 系统的核心模块之一，用于管理已确认购买意向的企业客户。客户可通过线索转化或手动创建两种方式产生，支持多联系人管理、客户档案 AI 生成、公海池管理等功能。

### 1.1 核心价值

| 特性 | 说明 |
|------|------|
| **360° 客户视图** | 客户信息、联系人、跟进记录、商机、合同、回款、发票一站式查看 |
| **AI 智能档案** | 自动生成企业背景、主营业务、公司官网等信息 |
| **公海池机制** | 支持客户退回公海、领取、分配，实现资源流转 |
| **多联系人管理** | 支持主联系人、决策人标记、组织架构关系 |
| **状态流转** | 跟进中 → 赢单/输单/失效，完整生命周期管理 |

### 1.2 模块定位

```
线索转化 → 客户创建 → 客户跟进 → 商机推进 → 合同签署
                ↓
           公海池流转（退回/领取/分配）
```

### 1.3 核心业务链路

```
线索模块 ──转化──→ 客户模块 ──创建商机──→ 商机模块
                      │                      │
                      ↓                      ↓
               联系人管理              合同签署
                      │                      │
                      └───────┬──────────────┘
                              ↓
                         回款/发票
```

## 二、业务生命周期（业务视角）

### 2.1 客户生命周期

```
潜在客户 → 跟进中 → 已成交 → 跟进中
              ↓
           已输单 → 已失效（公海池）
```

### 2.2 客户状态说明

| 状态值 | 名称 | 说明 |
|--------|------|------|
| `0` | 跟进中 (FOLLOWING) | 正常跟进状态 |
| `1` | 已赢单 (WON) | 成功成交，可创建合同 |
| `2` | 已输单 (LOST) | 失去客户，需记录输单原因 |
| `3` | 已失效 (INACTIVE) | 公海池客户或失效客户 |

### 2.3 客户来源

| 来源 | 说明 |
|------|------|
| **线索转化** | 从线索转化而来（主要来源） |
| **客户推荐** | 现有客户推荐 |
| **市场活动** | 通过市场活动获取 |
| **线上注册** | 客户在线注册 |
| **电话营销** | 电话营销获取 |
| **网站咨询** | 网站咨询获取 |
| **展会** | 展会获取 |
| **其他渠道** | 合作伙伴、其他来源 |

### 2.4 状态流转图

```
                ┌─────────────┐
                │   跟进中    │
                │    (0)      │
                └──────┬──────┘
                       │
       ┌───────────────┼───────────────┐
       │               │               │
       ↓               ↓               ↓
┌─────────────┐ ┌─────────────┐ ┌─────────────┐
│   已赢单    │ │   已输单    │ │   已失效    │
│    (1)      │ │    (2)      │ │    (3)      │
└──────┬──────┘ └─────────────┘ └──────┬──────┘
       │                              │
       ↓                              ↓
创建合同/商机                     公海池领取
       │                              │
       └──────────────────────────────┘
                       │
                       ↓
                ┌─────────────┐
                │   跟进中    │
                │    (0)      │
                └─────────────┘
```

### 2.5 状态变更场景

1. **跟进中 → 已成交**
   - 条件：客户首次签约
   - 操作：系统自动更新或手动标记
   - 结果：客户状态变为"已成交"

2. **跟进中 → 已输单**
   - 条件：客户选择竞品或明确表示不会采购
   - 操作：标记输单并填写原因
   - 结果：客户退回公海池

3. **已成交 → 跟进中**
   - 条件：已成交客户产生新的商机
   - 操作：创建新商机
   - 结果：客户状态保持活跃

4. **跟进中 → 已失效**
   - 条件：超过30天未跟进或明确表示暂时无需求
   - 操作：销售总监确认后退回公海
   - 结果：客户退回公海池

## 三、核心功能（合并）

### 3.1 客户创建

| 方式 | 说明 | 特点 |
|------|------|------|
| **线索转化** | 从合格线索转化为客户 | 自动创建主联系人、AI 生成档案、分析线索跟进记录 |
| **手动创建** | 直接创建新客户 | 需填写客户名称、城市等必填信息，AI 自动生成档案 |
| **批量导入** | 通过 Excel 批量导入 | 适用于批量数据迁移 |

**必填字段**：
- 客户公司名称
- 所在城市
- 所属行业

**可选字段**：
- 公司地址
- 公司规模
- 客户来源线索ID
- 负责人

**线索转化流程**：
1. 选择合格线索
2. 补充客户名称（可选，默认使用线索公司名）
3. 补充地址信息（可选）
4. 设置默认采购方式（可选）
5. 调用转化接口
6. 系统自动：创建客户 + 创建主联系人 + 触发 AI 档案生成 + 发送飞书通知

### 3.2 客户档案（AI 生成）

客户档案由 AI 自动生成，包含以下信息：

| 字段 | 说明 | 数据来源 |
|------|------|----------|
| `company_background` | 企业背景 | AI 根据客户名称搜索生成 |
| `company_website` | 公司官网 | AI 搜索获取 |
| `main_business` | 主营业务 | AI 分析生成 |
| `similar_customers` | 同行业客户 | AI 匹配现有客户库 |
| `project_background` | 项目需求背景 | 从线索跟进记录分析提取 |
| `industry` | 所属行业 | AI 推断（可手动修改） |

**档案状态**：
| 状态 | 说明 |
|------|------|
| `PENDING` | 等待生成（刚创建） |
| `GENERATING` | 正在生成（预计 10-30 秒） |
| `COMPLETED` | 生成完成 |
| `FAILED` | 生成失败（可重新生成） |

**档案生成流程**：
```
客户创建 → PENDING → 触发异步任务 → GENERATING
    ↓
AI 判断行业（如：金融、互联网）
    ↓
AI 搜索企业信息（背景、官网、主营业务）
    ↓
AI 匹配同行业客户（从现有客户库）
    ↓
AI 分析线索跟进记录（项目需求背景）
    ↓
COMPLETED（生成完成）
```

**重新生成档案**：
- 用户可手动触发重新生成（POST /{customer_id}/regenerate-profile）
- 生成失败时自动记录错误原因
- 档案内容可手动编辑覆盖

### 3.3 客户分配

**分配规则**：
1. **线索转化分配**：从线索转化时，客户归属原线索负责人
2. **手动分配**：销售总监可以手动分配客户给销售人员
3. **客户领取**：销售人员可以领取公海池中的客户

**分配权限**：
- 销售总监：可以分配任何客户给团队成员
- 销售成员：只能领取公海池客户，不能主动分配

**分配限制**：
- 每个销售人员最多拥有100个活跃客户
- 公海池中的客户可以被任何销售人员领取

### 3.4 联系人管理

| 功能 | 说明 |
|------|------|
| **添加联系人** | 姓名、手机号必填，支持职务、邮箱、微信等 |
| **主联系人设置** | 每个客户仅一个主联系人 |
| **决策人标记** | 标记关键决策人，影响销售策略 |
| **组织架构** | 通过 `reports_to` 字段建立汇报关系 |
| **联系人编辑/删除** | 支持修改联系人信息、删除联系人 |

**联系人字段**：
```python
name: str              # 姓名（必填）
gender: int            # 性别：0=未知, 1=男, 2=女
position: str          # 职务（如：CTO、采购经理）
is_decision_maker: bool # 是否关键决策人
mobile: str            # 手机号（必填）
email: str             # 邮箱
wechat_id: str         # 微信ID
remark: str            # 备注
reports_to: int        # 汇报对象联系人ID
is_primary: bool       # 是否主联系人
```

**联系人分类**：
- 主要联系人：主要的决策人
- 技术联系人：技术部门对接人
- 财务联系人：财务部门对接人
- 其他联系人：其他相关人员

### 3.5 客户跟进

**跟进要求**：
- 新客户3天内必须首次跟进
- 活跃客户至少每7天跟进一次
- 潜力客户至少每15天跟进一次

**跟进记录**：
- 跟进方式（电话、邮件、拜访、微信）
- 沟通内容摘要
- 客户反馈
- 下次跟进计划

**跟进提醒**：
- 超期未跟进自动提醒
- 计划跟进时间提醒

### 3.6 公海池管理

公海池用于存放无负责人的客户，实现资源流转。

| 功能 | 说明 | 权限要求 |
|------|------|----------|
| **退回公海** | 将客户退回公海池，解除负责人绑定 | 负责人本人或销售总监/管理员 |
| **领取客户** | 从公海池领取客户，成为负责人 | 所有销售人员 |
| **分配客户** | 管理员/总监将客户分配给指定人员 | TEAM_ADMIN 或 SALES_DIRECTOR |

**退回原因枚举**：
| 值 | 说明 |
|------|------|
| `丢单` | 客户已选择竞争对手 |
| `无意向` | 客户明确表示无合作意向 |
| `信息错误` | 客户信息不准确或无效 |
| `长期未跟进` | 超过规定时间未跟进 |
| `预算不足` | 客户预算无法匹配产品价格 |
| `其他` | 其他原因 |

**退回规则**：
1. **自动退回**：客户输单后自动退回公海
2. **手动退回**：销售总监可以将客户退回公海
3. **超期退回**：超过60天未跟进的客户自动退回公海

**退回影响**：
- 客户负责人清空
- 客户状态变为"已失效"
- 客户进入公海池，其他销售人员可以领取

### 3.7 客户领取

**领取权限**：
- 销售成员可以领取公海池客户
- 每天最多领取5个客户
- 领取后成为该客户的负责人

**领取限制**：
- 领取后7天内不能退回（除非客户明确拒绝）
- 领取后必须在3天内首次跟进

### 3.8 关联数据查询

客户详情页支持查看以下关联数据：

| Tab | 内容 |
|------|------|
| **跟进** | 跟进记录时间线，支持添加、编辑、删除 |
| **联系人** | 客户所有联系人 |
| **商机** | 客户关联的所有商机 |
| **合同** | 客户的所有合同 |
| **回款** | 客户的回款计划和回款记录 |
| **发票** | 客户的发票申请和发票抬头 |

## 六、前端页面（来自 MODULE）

### 6.1 客户列表页（Customers.vue）

**路径**：`/customers`

**功能**：
- 快捷筛选 Tab：所有客户、我的客户、公海客户
- 表格筛选：客户名称、城市、状态、行业
- 动态排序：创建时间、客户名称、城市、状态、行业
- 操作按钮：查看、编辑、新建商机、退回公海、赢单、输单、失效、删除
- 公海客户：领取操作

**权限控制**：
| 操作 | 权限要求 |
|------|----------|
| 新建客户 | `customer:create` |
| 编辑客户 | `customer:edit:all` 或 `customer:edit:own`（自己负责） |
| 删除客户 | `customer:delete:all` 或 `customer:delete:own`（自己负责） |
| 退回公海 | `customer:return` + 负责人本人或管理员/总监 |
| 领取客户 | `customer:view:own` 或 `customer:view:all` |

### 6.2 客户详情页（CustomerDetail.vue）

**路径**：`/customers/:id`

**功能**：
- 客户基本信息卡片：名称、状态、行业、规模、来源、城市、地址、负责人、采购方式
- 客户档案卡片：AI 生成的企业背景、官网、主营业务、同行业客户、项目背景
- Tab 切换：跟进、联系人、商机、合同、回款、发票
- 操作：编辑客户、重新生成档案

### 6.3 客户编辑页（CustomerEdit.vue）

**路径**：`/customers/:id/edit`

## 七、业务规则（合并）

### 7.1 客户名称唯一性

- 客户名称（`account_name`）全局唯一
- 创建/编辑时校验名称是否存在
- 归一化名称（`account_name_norm`）用于模糊匹配（去除后缀如"有限公司"、"股份有限公司"等）

### 7.2 负责人规则

- 创建客户时，未指定负责人则默认为创建人
- 退回公海时，解除负责人绑定（`owner_id` 设为 null）
- 领取/分配时，设置新的负责人

### 7.3 删除规则

- 逻辑删除，不物理删除
- 需检查权限：
  - 管理员可删除所有客户
  - 销售人员只能删除自己负责的客户

### 7.4 档案生成规则

- 客户创建后自动触发档案生成
- 档案生成异步执行，预计 10-30 秒
- 生成失败可重新触发
- 档案内容可手动编辑

### 7.5 公海流转规则

- 退回公海必须填写退回原因和详细说明
- 退回后发送飞书通知
- 领取客户后成为负责人
- 管理员/总监可强制分配客户

### 7.6 团队数据隔离

- 所有查询自动添加 `team_id` 过滤条件
- 跨团队数据不可见（包括公海池）
- API层自动注入当前用户的团队ID
- 联系人同样遵循团队隔离规则

### 7.7 转化关联规则

- 线索转化后记录 `source_lead_id` 关联原线索
- 删除客户时同步恢复线索状态为 NEW
- 客户的跟进记录继承线索的跟进历史

### 7.8 其他业务规则

1. **客户唯一性**：同一公司名称只能创建一个客户
2. **领养限制**：销售人员每天最多领取5个公海客户
3. **容量限制**：销售人员最多拥有100个活跃客户
4. **跟进要求**：新客户3天内必须首次跟进
5. **退回规则**：输单客户自动退回公海
6. **公海保护**：领取后7天内不能无故退回

## 十一、常见问题（来自 business）

**Q1: 客户输单后可以恢复吗？**

A: 输单后的客户退回公海池，可以重新领取，但历史记录会保留。

**Q2: 如何转移客户给其他销售人员？**

A: 目前只能通过销售总监重新分配客户，或先将客户退回公海再由其他人领取。

**Q3: 客户可以重新分配吗？**

A: 可以，销售总监有权将客户重新分配给团队成员，需要填写分配原因。

**Q4: 公海池的客户如何排序？**

A: 默认按客户退回公海的时间排序，最近退回的客户排在前面。

**Q5: 如何批量导入客户？**

A: 可以使用Excel模板批量导入客户，需要先下载模板，填写信息后上传。

**Q6: AI 档案生成失败怎么办？**

A: 可以手动重新触发生成（POST /{customer_id}/regenerate-profile），或手动编辑档案内容。
