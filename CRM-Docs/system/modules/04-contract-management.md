## 一、模块概述

合同管理模块用于管理从创建到履约完成的合同全生命周期。合同由赢单商机转化而来，是销售流程的关键产出物。合同创建后自动进入审批流程，审批通过后完成签署和生效，最终跟踪回款和履约状态。

**核心价值：**
- 审批流程自动化：合同创建后自动匹配审批流程并流转
- 合同编号规范化：系统自动生成唯一合同编号
- 标准单价计算：自动反算单价，保证定价一致性
- 回款状态跟踪：关联付款计划，实时更新回款进度

**业务定位：**
- 线索 → 客户 → 商机 → **合同** → 发票 → 收款
- 合同是销售转化的最终凭证，承接商机赢单，产出付款计划和发票申请

## 二、业务生命周期

### 2.1 合同状态流转

```
创建合同 → 草稿（DRAFT） → [自动审批流程] → 待审核（PENDING_REVIEW）
                          ↓
                      审批拒绝 → 草稿（可重新提交）
                          ↓
                      审批通过 → 已签署（SIGNED） → 已生效（EFFECTIVE）
                          ↓
                      已到期（EXPIRED）/ 已终止（TERMINATED）
```

**状态说明：**

| 状态值 | 枚举 | 说明 | 可操作 |
|--------|------|------|--------|
| DRAFT | 草稿 | 合同草稿，尚未提交审批 | 编辑、删除 |
| PENDING_REVIEW | 待审核 | 合同在审批流程中 | 查看详情、撤回审批 |
| SIGNED | 已签署 | 审批通过，完成签署 | 激活生效 |
| EFFECTIVE | 已生效 | 合同生效，进入履约期 | 添加付款计划、申请发票 |
| EXPIRED | 已到期 | 合同到期（订阅制到期或买断制期满） | 查看详情 |
| TERMINATED | 已终止 | 合同提前终止 | 查看详情 |

### 2.2 自动审批流程

**核心机制：** 合同创建后自动提交审批，无需手动操作

**流程匹配规则：**
- 系统根据合同金额、客户等信息自动匹配审批流程
- 不同金额区间匹配不同的审批流程节点
- 审批流程配置由系统管理员维护

**预置审批流程：**

**小额合同（< 10万）：**
```
提交 → 销售总监审批 → 通过 → 已签署
```

**中等金额合同（10-50万）：**
```
提交 → 销售总监审批 → 财务审批 → 通过 → 已签署
```

**大额合同（≥ 50万）：**
```
提交 → 销售总监审批 → 财务审批 → 系统管理员审批 → 通过 → 已签署
```

## 三、核心功能

### 3.1 合同创建

**创建方式：**

1. **从赢单商机创建（推荐）**
   - 路径：`POST /v1/contracts/from-opportunity/{opportunity_id}`
   - 参数：合同名称、签约联系人ID
   - 自动填充：客户、金额、用户数、授权模式等（从商机继承）

2. **手动创建**
   - 路径：`POST /v1/contracts`
   - 必填：合同名称、客户ID、商机ID、签约联系人、金额、用户数、授权模式、签约日期

**自动处理：**
- 合同编号：系统自动生成（格式：`HT-YYYYMMDD-序号`）
- 标准单价：自动反算（订阅制：总金额/用户数/年数；买断制：总金额/用户数/5）
- 到期日期：自动计算（生效日期 + 订阅年限）
- 审批流程：自动提交审批（创建后立即流转）
- 操作日志：自动记录创建事件

**必填字段：**
- 合同名称
- 关联客户
- 关联商机（必须为赢单状态）
- 签约联系人（必须属于该客户）
- 合同金额
- 采购用户数
- 授权模式
- 签约日期

### 3.2 合同草稿

**草稿状态：**
- 合同创建后默认为"草稿"状态
- 自动提交审批后状态变为"待审核"
- 草稿可以编辑和删除

**编辑权限：**
- 合同创建人可以编辑
- 系统管理员可以编辑
- 只能编辑草稿状态的合同

**编辑影响：**
- 修改金额/用户数 → 自动重新计算标准单价
- 修改授权模式/订阅年限 → 自动重新计算到期日期
- 不能修改客户ID、商机ID等核心关联

### 3.3 合同审批

**自动审批机制：**
- 合同创建后自动提交审批流程
- 系统匹配对应的审批流程并流转到第一个节点
- 发送飞书通知给第一级审批人

**审批操作：**

1. **审批通过**
   - 当前节点审批人点击通过
   - 流转到下一节点（如有）
   - 最终节点通过后合同状态变为"已签署"
   - 通知提交人审批完成

2. **审批拒绝**
   - 当前节点审批人点击拒绝
   - 合同状态退回"草稿"
   - 记录拒绝原因
   - 通知提交人被拒绝

3. **审批撤回**
   - 提交人可撤回审批中的合同
   - 合同状态退回"草稿"
   - 审批流程终止
   - 可修改后重新提交

**审批记录：**
- 所有审批操作都会记录
- 包含审批人、审批时间、审批意见
- 支持查询完整审批历史

### 3.4 合同签署

**签署操作：**
1. 审批通过后合同状态变为"已签署"
2. 填写实际签署日期
3. （可选）上传签署文件扫描件
4. 激活合同使其生效

**签署要求：**
- 必须经过审批流程
- 签署日期记录实际签约时间
- 文件格式支持：PDF、JPG、PNG（10MB以内）

### 3.5 合同生效

**生效操作：**
1. 合同签署后点击"激活生效"
2. 设置生效日期
3. 系统自动计算到期日期：
   - 订阅制：生效日期 + 订阅年限
   - 买断制：生效日期 + 5年
4. 合同状态变为"已生效"

**生效后：**
- 可添加付款计划（关联回款）
- 可申请发票（关联发票模块）
- 进入履约跟踪阶段
- 实时更新回款状态

### 3.6 合同履约

**履约跟踪：**
- 付款计划：设置分期付款时间和金额
- 回款记录：记录实际回款金额和时间
- 发票申请：申请开具发票
- 回款状态：自动更新（未回款/部分回款/已回款/逾期）

**回款状态说明：**

| 状态值 | 枚举 | 说明 |
|--------|------|------|
| UNPAID | 未回款 | 尚未回款 |
| PARTIAL | 部分回款 | 已回款部分金额 |
| COMPLETED | 已回款 | 全额回款完成 |
| OVERDUE | 逾期 | 付款计划逾期未回款 |

**合同完成：**
- 全额回款后合同进入完成状态
- 订阅制到期后合同自动变为"已到期"
- 买断制期满后合同自动变为"已到期"

### 3.7 合同终止

**终止场景：**
- 客户提前解约
- 退款取消合同
- 特殊情况终止

**终止操作：**
- 系统管理员可终止合同
- 需填写终止原因
- 通知相关人员
- 合同状态变为"已终止"

## 四、数据模型

### 4.1 合同表（crm_contracts）

**核心字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BigInteger | 主键 |
| team_id | BigInteger | 团队ID（团队隔离） |
| contract_number | String(50) | 合同编号（系统生成，唯一） |
| contract_name | String(255) | 合同名称 |
| customer_id | BigInteger | 关联客户ID |
| opportunity_id | BigInteger | 关联商机ID |
| signing_contact_id | BigInteger | 签约联系人ID |
| user_count | Integer | 采购用户数 |
| total_amount | Numeric(12,2) | 合同总金额 |
| license_type | String(20) | 授权模式：SUBSCRIPTION/PERPETUAL |
| subscription_years | Integer | 订阅年限（订阅制必填） |
| standard_unit_price | Numeric(10,2) | 标准单价（系统反算） |
| status | String(20) | 合同状态 |
| signing_date | Date | 签署日期 |
| effective_date | Date | 生效日期 |
| expiry_date | Date | 到期日期 |
| total_paid_amount | Numeric(12,2) | 累计已回款金额 |
| payment_status | String(20) | 回款状态 |
| creator_id | String(100) | 创建人飞书用户ID |

**关键索引：**
- idx_contract_customer（客户关联）
- idx_contract_opportunity（商机关联）
- idx_contract_status（状态筛选）
- idx_contract_number（编号查询）
- idx_contract_payment_status（回款状态）

**关联关系：**
- payment_plans：付款计划（一对多）
- invoice_applications：发票申请（一对多）

### 4.2 合同编号生成规则

**格式：** `HT-YYYYMMDD-序号`

**示例：**
- HT-20260612-001
- HT-20260612-002

**生成逻辑：**
- 每日重新计数
- 保证全局唯一
- 不可自定义修改

### 4.3 标准单价计算规则

**订阅制：**
```
标准单价 = 合同总金额 / 用户数 / 订阅年限
```

**买断制：**
```
标准单价 = 合同总金额 / 用户数 / 5
```

**计算时机：**
- 合同创建时自动计算
- 编辑金额/用户数/授权模式时自动重算

### 4.4 到期日期计算规则

**订阅制：**
```
到期日期 = 生效日期 + 订阅年限
```

**买断制：**
```
到期日期 = 生效日期 + 5年
```

#### 从商机创建合同（POST /v1/contracts/from-opportunity/{id})

**路径参数：**
- opportunity_id：商机ID（必须为赢单状态）

**Query 参数：**
- contract_name：合同名称（必填）
- signing_contact_id：签约联系人ID（必填，必须属于商机关联的客户）

**自动填充：**
- 客户ID：从商机关联客户获取
- 合同金额：从商机实际成交金额获取
- 用户数、授权模式、订阅年限：从商机继承

**业务逻辑：**
1. 验证商机为赢单状态（status=1）
2. 验证签约联系人属于该客户
3. 自动生成合同编号
4. 自动计算标准单价
5. 自动提交审批流程
6. 创建操作日志

#### 编辑合同（PUT /v1/contracts/{id})

**可编辑字段：**
- 合同名称
- 合同金额
- 用户数
- 签约日期
- 生效日期

**不可编辑字段：**
- 合同编号
- 客户ID
- 商机ID
- 签约联系人ID

**自动重算：**
- 金额/用户数变化 → 重算标准单价
- 生效日期/订阅年限变化 → 重算到期日期

## 七、业务规则

### 7.1 创建规则

1. **商机验证**：必须从赢单商机创建（status=1）
2. **联系人验证**：签约联系人必须属于商机关联的客户
3. **金额验证**：合同金额必须 > 0
4. **用户数验证**：采购用户数必须 > 0
5. **订阅验证**：订阅制必须填写订阅年限
6. **编号唯一**：合同编号系统生成，全局唯一
7. **自动审批**：创建后自动提交审批流程

### 7.2 编辑规则

1. **状态限制**：只能编辑草稿（DRAFT）状态的合同
2. **关联锁定**：不可修改客户ID、商机ID、签约联系人ID
3. **单价自动**：金额/用户数变化自动重算标准单价
4. **到期自动**：生效日期/订阅年限变化自动重算到期日期

### 7.3 删除规则

1. **状态限制**：只能删除草稿（DRAFT）状态的合同
2. **关联清理**：删除合同会级联删除付款计划和发票申请
3. **权限限制**：只有创建人或管理员可删除

### 7.4 审批规则

1. **自动提交**：创建后自动提交审批，无需手动操作
2. **流程匹配**：系统自动匹配审批流程（基于金额、客户等）
3. **顺序审批**：按节点顺序依次审批
4. **全员通过**：所有节点必须通过才算通过
5. **一票否决**：任一节点拒绝则整体拒绝
6. **撤回限制**：只有提交人可撤回审批中的合同

### 7.5 状态流转规则

1. **草稿 → 待审核**：创建时自动流转
2. **待审核 → 已签署**：审批流程通过后自动流转
3. **待审核 → 草稿**：审批拒绝或撤回后流转
4. **已签署 → 已生效**：手动激活生效
5. **已生效 → 已到期**：到期日期后自动流转
6. **已生效 → 已终止**：手动终止（管理员权限）

| contract:approve | 审批合同 | 销售总监、财务、管理员 |
| contract:activate | 激活生效 | 创建人或管理员 |
| contract:terminate | 终止合同 | 系统管理员 |

### 8.2 数据权限

**销售成员：**
- 查看自己创建的合同
- 创建合同（从赢单商机）
- 编辑自己的草稿合同
- 查看合同详情和审批进度
- 撤回审批中的合同

**销售总监：**
- 查看团队所有合同
- 审批合同（第一级审批节点）
- 查看合同统计
- 查看审批待办

**财务人员：**
- 审批合同（第二级审批节点）
- 查看合同金额和付款条款
- 审核发票申请

**系统管理员：**
- 查看所有团队合同
- 审批合同（第三级审批节点）
- 终止合同
- 管理审批流程配置
- 删除草稿合同（特殊情况）

### 8.3 团队隔离

所有合同操作强制校验 `team_id`：
- 创建时绑定当前用户团队
- 查询时过滤当前用户团队
- 编辑/删除时校验团队归属

## 十一、常见问题

**Q1: 合同创建后可以修改吗？**
A: 草稿状态可以修改，审批中的合同不能修改。审批拒绝或撤回后可重新编辑。

**Q2: 为什么合同创建后自动进入审批？**
A: 系统设计了自动审批机制，减少人工操作步骤。合同创建后自动匹配审批流程并流转，提高审批效率。

**Q3: 审批拒绝后怎么办？**
A: 审批拒绝后合同退回草稿状态，可以修改后重新提交审批。系统会记录拒绝原因供参考。

**Q4: 如何从商机创建合同？**
A: 在商机详情页（赢单状态）点击"创建合同"按钮，选择签约联系人后自动生成合同草稿。

**Q5: 合同编号可以自定义吗？**
A: 合同编号由系统自动生成（格式：HT-YYYYMMDD-序号），不支持自定义。编号全局唯一，便于查询和管理。

**Q6: 标准单价如何计算？**
A: 系统自动反算，订阅制：总金额/用户数/年数；买断制：总金额/用户数/5。修改金额或用户数会自动重算。

**Q7: 合同到期后会自动提醒吗？**
A: 系统会在到期日期前发送飞书通知提醒创建人。订阅制到期后合同状态自动变为"已到期"。

**Q8: 如何查看合同的回款进度？**
A: 在合同详情页查看付款计划列表和回款记录。系统实时更新回款状态（未回款/部分回款/已回款/逾期）。

**Q9: 合同可以删除吗？**
A: 只有草稿状态的合同可以删除。已审批或已生效的合同不可删除，只能查看历史记录。

**Q10: 审批流程如何配置？**
A: 审批流程由系统管理员配置，包括审批节点、审批角色、匹配规则。不同金额区间匹配不同的审批流程。
