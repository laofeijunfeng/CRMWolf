# CRM系统业务流程概览

## 1. 系统概述

本CRM系统是一个完整的客户关系管理平台，涵盖从线索获取到合同签署的全流程管理。系统支持销售团队管理客户、跟进商机、签署合同，并提供完善的审批流程和数据统计功能。

## 2. 核心业务模块

### 2.1 线索管理（Leads）
线索是销售流程的起点，代表潜在的商业机会。

**主要功能：**
- 线索创建和录入
- 线索分配和领取
- 线索跟进和记录
- 线索转化为客户
- 线索输单和公海管理

**关键指标：**
- 线索转化率
- 线索跟进率
- 平均转化周期

**详细说明：** [01-lead-management.md](./01-lead-management.md)

### 2.2 客户管理（Customers）
客户是企业或组织，是销售活动的核心对象。

**主要功能：**
- 客户信息管理
- 客户分配和领取
- 客户跟进和记录
- 客户联系人管理
- 客户状态管理
- 公海池管理

**关键指标：**
- 客户成交率
- 客户活跃度
- 客户价值分析

**详细说明：** [02-customer-management.md](./02-customer-management.md)

### 2.3 商机管理（Opportunities）
商机是与客户的具体商业机会，关联到具体的项目和金额。

**主要功能：**
- 商机创建和管理
- 销售阶段推进
- 商机赢单和输单
- 销售漏斗管理
- 业绩预测

**关键指标：**
- 商机赢单率
- 平均销售周期
- 销售漏斗转化率

**详细说明：** [03-opportunity-management.md](./03-opportunity-management.md)

### 2.4 合同管理（Contracts）
合同是与客户签署的正式协议，标志销售成功。

**主要功能：**
- 合同创建和审批
- 多级审批流程
- 合同签署和归档
- 合同履行跟踪
- 合同统计分析

**关键指标：**
- 合同审批通过率
- 平均审批时长
- 合同总金额

**详细说明：** [04-contract-management.md](./04-contract-management.md)

### 2.5 回款管理（Payments）
回款管理是合同履行的最终环节，直接关系到企业的现金流健康。

**主要功能：**
- 回款计划制定（分期回款）
- 回款记录登记
- 回款进度跟踪
- 回款汇总统计
- 逾期预警提醒

**关键指标：**
- 回款完成率
- 平均回款周期
- 逾期回款金额

**详细说明：** [05-payment-management.md](./05-payment-management.md)

### 2.6 发票管理（Invoices）
发票管理是实现业财一体化的关键环节，规范开票流程并与回款关联。

**主要功能：**
- 开票抬头维护
- 发票申请创建
- 发票审批流程
- 发票开具跟踪
- 发票关联回款

**关键指标：**
- 发票申请总数
- 待审批发票数
- 本月开票金额
- 开票完成率

**详细说明：** [06-invoice-management.md](./06-invoice-management.md)

### 2.7 财务管理（Finance）
财务管理是资金流与票据流的管理核心，负责全流程财务合规性监督。

**主要功能：**
- 回款确认入账
- 应收账款账龄分析
- 逾期回款预警
- 财务报表统计
- 风险监控评估

**关键指标：**
- 应收账款总额
- 逾期金额
- 逾期率
- 坏账风险率
- 回款确认率

**详细说明：** [07-finance-management.md](./07-finance-management.md)

## 3. 业务流程全景

### 3.1 标准销售流程

```
┌─────────┐    转化    ┌─────────┐    创建    ┌─────────┐
│  线索   │ ────────→  │  客户   │ ────────→  │  商机   │
└─────────┘           └─────────┘           └─────────┘
     │                                           │
     │ 跟进                                      │ 推进阶段
     ↓                                           ↓
  添加跟进记录                                更新赢率
     │                                           │
     │ 输单/转化                                 │ 赢单
     ↓                                           ↓
  退回公海                                   ┌─────────┐
                                              │  合同   │
                                              └─────────┘
                                                   │
                                                   │ 审批
                                                   ↓
                                                签署完成
```

### 3.2 完整业务时序

**阶段1：线索获取**
1. 通过各种渠道获取线索（线上注册、市场活动等）
2. 分配给销售人员跟进
3. 销售人员添加跟进记录

**阶段2：线索转化**
1. 确认客户有真实需求
2. 与决策人建立联系
3. 将线索转化为客户
4. 自动创建初始商机

**阶段3：商机跟进**
1. 按销售阶段推进商机
2. 更新商机赢率
3. 记录跟进过程
4. 预测成交时间

**阶段4：赢单签约**
1. 商机赢单
2. 创建合同
3. 提交审批流程
4. 审批通过后签署

**阶段5：合同履行与回款**
1. 上传签署文件
2. 创建回款计划（分期）
3. 跟踪合同履行
4. 登记回款记录
5. 回款汇总统计
6. 完成合同

## 4. 数据流转关系

### 4.1 线索 → 客户

**触发条件：** 线索转化

**数据传递：**
- 客户公司名称 = 线索公司名称
- 客户所属城市 = 线索所属城市
- 客户所属行业 = 线索所属行业
- 客户负责人 = 线索负责人

**状态变化：**
- 线索状态 → 已转化
- 创建新客户记录

### 4.2 客户 → 商机

**触发条件：** 创建商机

**关联关系：**
- 商机必须关联客户
- 一个客户可以有多个商机

**数据关联：**
- 商机客户ID = 客户ID
- 商机负责人 = 客户负责人

### 4.3 商机 → 合同

**触发条件：** 商机赢单后创建合同

**数据传递：**
- 合同客户ID = 商机客户ID
- 合同金额 = 商机实际成交金额
- 合同负责人 = 商机负责人
- 合同商机ID = 商机ID（可选）

**状态变化：**
- 商机状态 = 已赢单
- 创建新合同记录（草稿）

### 4.4 合同 → 回款计划

**触发条件：** 合同签署后创建回款计划

**关联关系：**
- 一个合同可以有多个回款计划（分期）
- 回款计划必须关联合同

**数据关联：**
- 回款计划合同ID = 合同ID
- 回款计划总额 ≤ 合同总金额
- 登记回款后自动更新合同回款状态

## 5. 角色与权限

### 5.1 销售成员（Sales Member）

**数据权限：**
- 只能查看自己创建的数据
- 可以查看公海池数据
- 可以领取公海池数据

**操作权限：**
- 创建线索、客户、商机、合同
- 编辑自己的数据
- 推进商机阶段
- 提交合同审批

### 5.2 销售总监（Sales Director）

**数据权限：**
- 查看团队所有数据
- 查看团队统计报表

**操作权限：**
- 分配线索、客户给团队成员
- 审批合同
- 退回数据到公海
- 查看销售漏斗

### 5.3 系统管理员（System Admin）

**数据权限：**
- 查看所有数据
- 导出数据报表

**操作权限：**
- 管理用户和角色
- 配置系统参数
- 管理审批流程
- 取消合同

### 5.4 财务角色（Finance）

**数据权限：**
- 查看全公司所有客户、合同、回款计划及发票数据
- 查看所有财务分析报表
- 审计所有用户的财务相关操作日志

**操作权限：**
- 发票申请审批（批准/拒绝）
- 回款记录确认入账
- 查看应收账款账龄分析
- 查看逾期回款预警
- 生成财务报表

## 6. 关键业务规则

### 6.1 数据唯一性
- 同一电话号码只能创建一个线索
- 同一公司名称只能创建一个客户
- 合同编号全局唯一

### 6.2 数据分配
- 线索分配后归属销售人员
- 客户可以通过分配或领取获取
- 公海池数据可以被任何人领取

### 6.3 状态流转
- 线索转化后不能恢复
- 商机阶段按顺序推进
- 合同必须经过审批流程

### 6.4 权限控制
- 销售成员只能操作自己的数据
- 销售总监可以操作团队数据
- 系统管理员拥有所有权限

### 6.5 数据关联
- 商机必须关联客户
- 合同建议关联商机
- 跟进记录关联具体对象

## 10. 常见问题

**Q1: 线索和客户有什么区别？**
A: 线索是潜在的商业机会，可能成为客户；客户是已经建立联系的企业或组织。线索转化后成为客户。

**Q2: 商机和合同有什么区别？**
A: 商机是具体的销售机会，处于跟进阶段；合同是已经签署的正式协议，标志销售成功。商机赢单后创建合同。

**Q3: 公海池是什么？**
A: 公海池是存放无负责人线索和客户的地方，可以被任何销售人员领取。输单或失效的数据会退回公海。

**Q4: 如何提高线索转化率？**
A: 及时跟进、详细记录、准确判断客户需求、精准转化有效线索。

**Q5: 审批流程可以自定义吗？**
A: 系统预置了三个审批流程，系统管理员可以配置新的审批流程。

**Q6: 如何创建回款计划？**
A: 合同签署后，可以为合同创建分期回款计划，系统会自动跟踪回款进度。

**Q7: 回款记录可以修改吗？**
A: 可以。修改回款记录后，系统会自动重新计算相关金额和状态。
