# 商机管理流程

## 1. 商机生命周期

```
创建商机 → 跟进中 → 各销售阶段推进 → 已赢单
                                    ↓
                                 已输单
```

## 2. 商机状态说明

| 状态 | 值 | 说明 |
|------|-----|------|
| 跟进中 | FOLLOWING | 正在跟进的商机 |
| 已赢单 | WON | 成功赢单的商机 |
| 已输单 | LOST | 确认无法成交的商机 |

## 3. 销售阶段

### 3.1 预置销售阶段

系统预置了5个销售阶段，按顺序推进：

| 阶段 | 赢率 | 说明 | 典型时长 |
|------|------|------|----------|
| 初步接触 | 10% | 首次与客户建立联系，了解基本情况 | 1-3天 |
| 需求分析 | 25% | 深入了解客户需求，明确痛点 | 3-7天 |
| 方案报价 | 50% | 提供解决方案和报价方案 | 5-10天 |
| 谈判审核 | 75% | 商务谈判和合同审核阶段 | 7-14天 |
| 赢单 | 100% | 成功签约，项目启动 | - |

### 3.2 阶段推进规则

1. **顺序推进**：必须按顺序推进，不能跳过阶段
2. **不可回退**：已推进的阶段不能回退（特殊情况下管理员可操作）
3. **推进记录**：每次推进需要记录推进原因
4. **停留时间**：建议在每个阶段停留不超过预期时间

## 4. 核心功能

### 4.1 商机创建

**创建方式：**
- **线索转化**：从线索转化时自动创建商机
- **手动创建**：销售人员手动创建商机

**必填字段：**
- 商机名称（项目名称）
- 关联客户
- 预计总金额
- 采购用户数
- 授权模式（订阅/买断）
- 采购类型（新购/续购/增购）
- 预计成交日期
- 销售阶段

**自动计算字段：**
- 标准单价：系统自动计算
  - 订阅制：标准单价 = 总金额 / 用户数 / 年数
  - 买断制：标准单价 = 总金额 / 用户数 / 5

### 4.2 商机跟进

**跟进要求：**
- 新商机2天内必须首次跟进
- 每个阶段至少跟进一次
- 预计成交日期前7天必须重点跟进

**跟进内容：**
- 客户需求确认
- 竞品情况分析
- 决策流程了解
- 预算和采购时间
- 关键决策人识别

**阶段推进判断：**
- 初步接触 → 需求分析：已与客户建立联系，了解基本需求
- 需求分析 → 方案报价：已明确客户痛点和需求，可以提供方案
- 方案报价 → 谈判审核：已提交方案，客户进入商务谈判
- 谈判审核 → 赢单：已签署合同或收到定金

### 4.3 商机推进

**推进操作：**
- 销售人员主动推进到下一阶段
- 需要填写推进说明
- 系统记录推进时间和操作人

**推进影响：**
- 商机阶段更新
- 赢率自动更新为阶段对应的赢率
- 创建阶段变更记录

**特殊情况：**
- 超过预计成交日期仍未赢单，需要说明原因
- 长期停留在同一阶段（超过30天）需要关注

### 4.4 商机赢单

**赢单条件：**
- 客户已签署合同
- 或已收到定金/预付款
- 或有明确的采购承诺

**赢单操作：**
1. 标记商机为"已赢单"
2. 填写实际成交金额
3. 填写实际成交日期
4. 系统自动：
   - 将商机阶段更新为"赢单"
   - 将赢率设置为100%
   - 关联客户状态更新为"已成交"

**赢单后：**
- 通知相关人员（销售总监）
- 创建合同记录
- 进入项目交付流程

### 4.5 商机输单

**输单原因：**
- 客户预算不足
- 客户选择了竞品
- 客户暂时没有需求
- 产品功能不符合需求
- 价格过高
- 其他原因

**输单操作：**
1. 标记商机为"已输单"
2. 选择输单原因
3. 填写详细说明
4. 系统记录输单信息

**输单后：**
- 通知销售总监
- 商机状态锁定，不能再修改
- 保留完整历史记录

### 4.6 商机关闭

**关闭规则：**
- 赢单或输单后商机自动关闭
- 关闭后商机信息只读，不可修改
- 可以查看完整历史记录

## 5. 商机信息管理

**基本信息：**
- 商机名称（项目名称）
- 关联客户
- 预计总金额
- 实际成交金额（赢单后）
- 采购用户数
- 标准单价
- 授权模式
- 订阅年限（订阅制）

**销售信息：**
- 采购类型
- 决策人数
- 预计成交日期
- 实际成交日期（赢单后）
- 当前阶段
- 当前赢率

**负责人信息：**
- 商机负责人
- 商机创建人
- 阶段变更历史

## 6. 权限控制

### 6.1 销售成员权限

- 查看自己的商机
- 创建商机
- 推进商机阶段
- 标记商机赢单/输单
- 编辑商机信息（跟进中状态）
- 查看商机统计

### 6.2 销售总监权限

- 查看团队所有商机
- 查看商机统计报表
- 查看销售漏斗
- 查看团队业绩
- 查看商机详情

### 6.3 系统管理员权限

- 查看所有商机
- 管理销售阶段
- 商机数据统计
- 导出商机报表

## 7. 数据统计

### 7.1 销售漏斗

统计各阶段的商机数量和金额：

```
阶段          商机数量    总金额    平均金额    平均赢率
初步接触         15        450万     30万       10%
需求分析         10        500万     50万       25%
方案报价          8        600万     75万       50%
谈判审核          5        400万     80万       75%
赢单             12        720万     60万      100%
```

### 7.2 赢率分析

- 各阶段实际赢单率
- 预计赢单金额 vs 实际赢单金额
- 赢单周期分析

### 7.3 输单分析

- 输单原因分布
- 竞品分析
- 输单商机特征分析

### 7.4 阶段耗时分析

- 各阶段平均停留时间
- 超期商机预警
- 阶段转化效率

## 8. 业务规则

1. **阶段顺序**：必须按预定义的顺序推进销售阶段
2. **金额限制**：预计总金额必须大于0
3. **用户数限制**：采购用户数必须大于0
4. **订阅年限**：订阅制必须有订阅年限
5. **预计成交日期**：不能早于创建日期
6. **赢单验证**：赢单时必须填写实际成交金额和日期
7. **输单原因**：输单时必须选择输单原因
8. **关闭规则**：赢单或输单后商机自动关闭

## 9. 飞书通知集成

**通知场景：**
- 商机创建通知：通知销售总监有新商机创建
- 阶段推进通知：通知销售总监商机阶段变更
- 赢单通知：通知相关人员商机成功赢单
- 输单通知：通知销售总监商机输单
- 超期提醒：提醒销售人员商机超期未推进

## 10. 常见问题

**Q1: 商机可以跳过销售阶段吗？**
A: 不能跳过，必须按顺序推进每个阶段。这样可以更好地跟踪销售进度。

**Q2: 商机赢单后可以修改信息吗？**
A: 赢单后商机信息只读，不能修改。如需修改，需要特殊权限。

**Q3: 如何调整销售阶段？**
A: 销售阶段由系统管理员配置，普通用户不能修改。

**Q4: 商机输单后可以重新激活吗？**
A: 输单后不能重新激活，但可以创建新的商机。

**Q5: 预计成交日期过期了怎么办？**
A: 需要及时更新预计成交日期，并说明延期原因。
