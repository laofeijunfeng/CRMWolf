# 回款计划与回款管理流程

## 1. 模块概述

回款管理是合同履行的最终环节，直接关系到企业的现金流健康。本模块提供对合同回款的**精细化、自动化、全生命周期**管理能力，支持制定分期回款计划、跟踪回款进度、自动计算汇总，并在关键节点提供提醒预警。

### 1.1 核心价值

- **计划化回款**：为合同制定清晰的分期回款计划，明确每笔款项的时间和金额
- **自动化跟踪**：系统自动计算回款进度，无需人工统计
- **实时汇总**：实时更新合同累计回款金额和回款状态
- **逾期预警**：自动检测逾期回款，及时提醒相关人员
- **完整审计**：记录每笔回款的详细信息，形成完整的审计线索

### 1.2 适用场景

- **分期回款合同**：支持按进度分阶段收款（如首付款、中期款、尾款）
- **大额合同管理**：对大额合同进行分期回款跟踪
- **回款催收**：通过逾期提醒功能辅助催收工作
- **财务分析**：提供准确的回款数据支持财务决策

## 2. 核心业务流程

```
合同已签署/已生效
    ↓
创建回款计划（支持分期）
    ↓
系统自动跟踪回款进度
    ↓
到期前提醒 → 登记回款 → 状态自动更新
    ↓                  ↓
逾期预警          回款汇总统计
```

## 3. 回款计划管理

### 3.1 回款计划生命周期

| 状态 | 值 | 说明 | 可执行操作 |
|------|-----|------|----------|
| 待回款 | PENDING | 尚未到期的回款计划 | 登记回款、修改计划 |
| 已逾期 | OVERDUE | 超过计划回款日期且未完成 | 登记回款、修改计划 |
| 部分回款 | PARTIAL | 已收到部分款项 | 继续登记回款 |
| 已完成 | COMPLETED | 已全部收回 | 查看详情、查看记录 |

### 3.2 创建回款计划

**前提条件：**
- 合同状态为"已签署"（SIGNED）或"已生效"（EFFECTIVE）
- 合同尚未创建回款计划，或需要补充新的回款计划

**创建方式：**
- **批量创建**：一次性创建多个阶段的回款计划
- **分期设置**：支持设置不同的回款阶段（首付款、中期款、尾款等）

**数据校验：**
- 所有阶段的计划金额之和 ≤ 合同总金额
- 每个阶段的计划金额必须大于0
- 必须指定计划回款日期

**创建示例：**
```
合同金额：250,000元
回款计划：
  - 首付款：75,000元（30%），2025-02-15
  - 中期款：100,000元（40%），2025-03-10
  - 尾款：75,000元（30%），2025-04-09
```

### 3.3 修改回款计划

**修改限制：**
- ❌ 已完成的回款计划不能修改
- ❌ 已有回款记录的计划不能修改金额和日期
- ✅ 可以修改待回款和已逾期状态的计划（前提是无回款记录）

**可修改字段：**
- 阶段名称（stage_name）
- 计划金额（planned_amount）
- 计划回款日期（due_date）
- 备注（notes）

### 3.4 删除回款计划

**删除条件：**
- ❌ 存在关联的回款记录时不能删除
- ✅ 无回款记录的计划可以删除

## 4. 回款记录管理

### 4.1 登记回款

**业务场景：**
收到客户付款后，需要在系统中登记回款记录。

**数据校验：**
- 单次回款金额必须大于0
- 累计回款金额不得超过计划金额
- 必须指定实际回款日期

**自动处理逻辑：**
1. 创建回款记录
2. 累加到合同累计回款金额
3. 计算该计划的累计回款金额
4. 更新回款计划状态：
   - 累计回款 ≥ 计划金额 → COMPLETED（已完成）
   - 累计回款 > 0 → PARTIAL（部分回款）
   - 未回款且已逾期 → OVERDUE（已逾期）
   - 未回款且未到期 → PENDING（待回款）
5. 更新合同回款状态：
   - 所有计划已完成 → COMPLETED（已回完）
   - 存在逾期计划 → OVERDUE（有逾期）
   - 部分回款 → PARTIAL（部分回款）
   - 无回款 → UNPAID（未回款）

**登记信息：**
- 实际回款金额（actual_amount）
- 实际回款日期（payment_date）
- 回款凭证附件（proof_attachment）- 可选
- 备注（notes）- 可选

### 4.2 修改/删除回款记录

**业务场景：**
- 登记错误需要修正
- 需要删除错误的回款记录

**自动处理逻辑：**
修改或删除回款记录后，系统会自动：
1. 重新计算该计划的累计回款金额
2. 更新回款计划状态
3. 重新计算合同的累计回款总额
4. 更新合同回款状态

**数据一致性保证：**
- 所有金额计算在数据库事务中完成
- 确保金额和状态的准确性

## 5. 合同回款状态

### 5.1 回款状态定义

| 状态 | 值 | 说明 | 触发条件 |
|------|-----|------|----------|
| 未回款 | UNPAID | 尚未收到任何回款 | 所有回款计划都未回款 |
| 部分回款 | PARTIAL | 已收到部分款项 | 存在回款记录但未全部完成 |
| 已回完 | COMPLETED | 全部款项已收齐 | 所有回款计划都已完成 |
| 有逾期 | OVERDUE | 存在逾期未回款 | 存在逾期的回款计划 |

### 5.2 状态自动更新

合同回款状态由系统自动计算和更新，无需手动维护：

```
初始化：UNPAID（未回款）
  ↓
登记首笔回款 → PARTIAL（部分回款）
  ↓
继续回款
  ↓
全部完成 → COMPLETED（已回完）

任何时间出现逾期 → OVERDUE（有逾期）
```

## 6. 回款汇总与统计

### 6.1 合同回款汇总

提供单个合同的完整回款统计信息：

**汇总指标：**
- 合同总额（total_amount）
- 累计已回款金额（total_paid_amount）
- 合同回款状态（payment_status）
- 回款计划总数（payment_plans_count）
- 已完成计划数（completed_plans_count）
- 逾期计划数（overdue_plans_count）
- 待回款金额（remaining_amount）

**数据实时性：**
- 所有数据实时计算，准确反映当前回款状况
- 登记回款后立即更新汇总数据

### 6.2 回款进度分析

**计划维度分析：**
- 每个计划的回款进度
- 已回款金额 vs 计划金额
- 回款完成率

**时间维度分析：**
- 按月统计回款金额
- 按季度统计回款趋势
- 回款周期分析

## 7. 提醒与预警

### 7.1 即将到期提醒

**查询范围：**
- 可自定义天数范围（如未来7天、30天）
- 仅查询待回款状态的计划

**提醒内容：**
- 合同名称
- 回款阶段名称
- 计划回款金额
- 计划回款日期
- 距离到期天数

**使用场景：**
- 提前通知销售人员跟进回款
- 准备收款相关工作

### 7.2 逾期预警

**查询条件：**
- 回款日期已过
- 回款计划状态不为"已完成"

**预警内容：**
- 逾期合同信息
- 逾期阶段名称
- 逾期金额
- 逾期天数（负数表示）

**使用场景：**
- 逾期回款催收
- 风险评估
- 管理层报告

### 7.3 飞书通知集成

**通知触发：**
- 定时任务自动检测
- 每日早上检测即将到期和已逾期的回款

**通知对象：**
- 合同负责人/创建人
- 销售总监

**通知内容：**
- 回款提醒通知
- 逾期预警通知

### 8.1 操作权限

| 操作 | 销售成员 | 销售总监 | 系统管理员 |
|------|---------|---------|-----------|
| 查看回款计划 | 自己的合同 | 团队所有 | 全部 |
| 创建回款计划 | 自己的合同 | 团队所有 | 全部 |
| 修改回款计划 | 自己的合同 | 团队所有 | 全部 |
| 删除回款计划 | 自己的合同 | 团队所有 | 全部 |
| 登记回款 | 自己的合同 | 团队所有 | 全部 |
| 修改/删除回款记录 | 自己的合同 | 团队所有 | 全部 |
| 查看回款统计 | 自己的数据 | 团队数据 | 全部 |

### 8.2 数据权限

- **销售成员**：只能查看和操作自己负责的合同的回款数据
- **销售总监**：可以查看和操作团队所有合同的回款数据
- **系统管理员**：拥有全部数据权限

## 9. 业务规则

### 9.1 金额规则

1. **计划总额限制**：所有回款计划的金额之和不得超过合同总金额
2. **单笔回款限制**：单次登记的回款金额不能超过该计划的待回款金额
3. **金额精度**：所有金额字段保留两位小数

### 9.2 状态规则

1. **计划状态自动更新**：登记回款后自动更新计划状态
2. **合同状态自动更新**：任何回款操作都会重新计算合同回款状态
3. **状态优先级**：逾期（OVERDUE）> 已完成（COMPLETED）> 部分回款（PARTIAL）> 未回款（UNPAID）

### 9.3 操作规则

1. **已完成计划锁定**：已完成的回款计划不能修改和删除
2. **有记录计划限制**：已有回款记录的计划不能修改金额和日期
3. **删除限制**：存在回款记录的计划不能删除

## 10. 常见问题

**Q1: 回款计划可以创建多少个阶段？**
A: 不限制阶段数量，但所有阶段的计划金额之和不能超过合同总金额。

**Q2: 如果实际回款金额超过计划金额怎么办？**
A: 系统会拒绝超额登记，必须先修改回款计划的计划金额，或者创建新的回款计划。

**Q3: 登记回款后发现错误，可以修改吗？**
A: 可以。修改回款记录后，系统会自动重新计算相关金额和状态。

**Q4: 合同回款状态是手动维护的吗？**
A: 不是。合同回款状态由系统根据所有回款计划的完成情况自动计算和维护。

**Q5: 如何查看逾期的回款？**
A: 可以通过"查询逾期回款"接口，或查看回款汇总中的"逾期计划数"指标。

**Q6: 回款提醒是自动发送的吗？**
A: 回款提醒接口提供了查询功能，可以通过定时任务定期查询并通过飞书发送通知。

**Q7: 删除回款记录后，之前的金额会重新计算吗？**
A: 会。删除回款记录后，系统会自动重新计算该计划的累计回款金额和合同的回款状态。

## 11. 最佳实践

### 11.1 回款计划制定

1. **合理分期**：根据项目进度和客户约定合理设置分期
2. **预留缓冲**：考虑实际业务情况，预留适当的回款缓冲时间
3. **明确阶段**：每个阶段名称应该清晰描述该阶段的业务含义

### 11.2 回款跟进

1. **提前提醒**：在回款到期前3-7天开始提醒客户
2. **及时登记**：收到款项后及时在系统中登记
3. **逾期处理**：对逾期回款及时跟进并记录原因

### 11.3 数据管理

1. **定期核对**：定期核对系统回款数据与财务账目
2. **凭证留存**：上传银行回单等凭证附件
3. **完整记录**：详细记录每笔回款的相关信息

## 12. 集成关系

### 12.1 与合同模块集成

- **合同状态校验**：只有已签署或已生效的合同可以创建回款计划
- **合同金额限制**：回款计划总额受合同总金额限制
- **双向关联**：回款计划关联到合同，合同也关联所有回款计划

### 12.2 与飞书集成

- **用户认证**：回款记录关联飞书用户信息
- **消息通知**：支持通过飞书发送回款提醒和逾期预警
- **用户信息**：登记人信息来自飞书用户数据

### 12.3 数据统计集成

- **销售业绩**：回款金额作为销售业绩的重要指标
- **财务报表**：回款数据用于财务分析和报表
- **预测分析**：基于回款计划进行现金流预测
