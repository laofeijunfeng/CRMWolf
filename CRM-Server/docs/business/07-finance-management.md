# 财务管理模块

## 1. 模块概述

### 1.1 产品背景

财务管理是CRM系统中**资金流与票据流的管理核心**，负责监督从合同签订到回款完结的全流程财务合规性。该模块与销售、管理流程紧密衔接，确保业务数据能够自动、准确地转化为财务数据。

### 1.2 核心价值

- **业财一体化**：打通业务（合同、回款）与财务（发票、对账）流程，消除数据隔阂
- **风险可控化**：通过审批流、额度控制与逾期监控，有效降低坏账与合规风险
- **效率提升**：自动化票据审批、收款匹配及报表生成，大幅减少财务人员手工操作

## 2. 角色定位与权限

### 2.1 财务角色定位

财务角色是CRM系统中负责资金与票据管理的专业角色，主要职责包括：

- **发票审批**：对销售提交的发票申请进行财务审批
- **回款确认**：确认销售登记的回款记录，确保资金准确入账
- **风险监控**：通过账龄分析和逾期预警，识别潜在的财务风险
- **报表分析**：为管理层提供准确的财务数据分析

### 2.2 权限模型

| 权限范围 | 授权内容 | 说明与约束 |
| :--- | :--- | :--- |
| **数据查看** | 可查看**全公司**所有客户、合同、回款计划及发票数据 | 远大于销售角色的权限，便于全局财务审核与审计 |
| **发票管理** | 拥有所有发票申请的**审批权**（APPROVE/REJECT） | 核心权限之一 |
| **回款确认** | 对销售登记的回款记录具有**确认入账**的最终权限 | 确保资金准确无误 |
| **财务报表** | 可访问所有财务分析报表（如应收账款账龄分析表） | 普通销售成员无权查看 |
| **操作日志** | 可审计所有用户的财务相关操作日志 | 满足合规与审计要求 |

### 2.3 角色代码

系统使用以下角色代码进行权限控制：

- `admin` - 系统管理员
- `finance` - 财务角色
- `sales_director` - 销售总监
- `sales_member` - 销售成员

## 3. 核心功能模块

### 3.1 回款确认流程

财务人员需要对销售登记的"预计回款"进行实质性的"确认入账"，这是风险控制的关键步骤。

#### 业务流程

```
销售登记回款
    ↓
状态：待确认（PENDING）
    ↓
财务核对银行到账信息
    ↓
财务确认入账
    ↓
状态：已确认（CONFIRMED）
```

#### 确认规则

1. **数据核对**：财务人员核对系统内的回款记录与银行实际到账信息
2. **执行确认**：确认后回款记录被标记为 CONFIRMED（已确认）状态
3. **关联发票**：支持在确认时关联已审批通过的发票，实现票款匹配
4. **争议处理**：如发现数据不符，可标记为 DISPUTED（有争议）状态

#### 数据模型增强

回款记录表（crm_payment_records）新增以下字段：

| 字段名 | 类型 | 描述 |
|--------|------|------|
| confirmation_status | varchar(20) | 确认状态：PENDING（待确认），CONFIRMED（已确认），DISPUTED（有争议） |
| confirmed_by | varchar(100) | 确认人（财务用户ID） |
| confirmed_by_name | varchar(100) | 确认人姓名 |
| confirmed_time | datetime | 确认入账时间 |
| confirmation_notes | text | 确认备注 |

### 3.2 应收账款管理

该功能为财务角色提供主动的风险管理能力。

#### 账龄分析

**功能描述**：按时间区间统计逾期应收账款金额，帮助财务评估坏账风险。

**账龄区间**：
- 0-30天：正常逾期
- 31-60天：关注逾期
- 61-90天：高风险逾期
- 90天以上：严重逾期

**分析维度**：
- 总逾期金额
- 逾期计划数量
- 各账龄区间的金额分布
- 详细逾期明细（合同、客户、逾期天数）

#### 逾期预警

**功能描述**：获取所有逾期回款计划列表，用于催收管理。

**筛选条件**：
- 最小逾期天数
- 最大逾期天数
- 最小欠款金额

**输出信息**：
- 合同信息
- 客户信息
- 回款阶段
- 逾期金额和天数
- 负责人信息

### 3.3 财务报表与分析

为管理层决策提供数据支持。

#### 合同收入统计报表

**功能描述**：按时间、产品、客户维度统计合同收入、实收金额、欠款金额等。

**分组方式**：
- `day` - 按天统计
- `week` - 按周统计
- `month` - 按月统计
- `customer` - 按客户统计

**统计指标**：
- 合同数量
- 合同总金额
- 实收金额
- 欠款金额

## 5. 业务规则

### 5.1 发票审批规则

1. **审批权隔离**：发票的创建者（销售）与审批者（财务）必须为不同人员，系统应在后端强制校验
2. **状态联动**：审批通过后，发票申请状态变为 APPROVED，并触发通知给申请人
3. **记录审计**：记录审批人、审批时间及意见，确保流程可追溯

### 5.2 回款确认规则

1. **金额校验**：关联发票的总金额不能超过回款金额
2. **状态限制**：只能确认待确认（PENDING）状态的回款记录
3. **发票校验**：关联的发票必须属于同一客户和合同

### 5.3 逾期计算规则

1. **自动计算**：应收账款账龄分析由系统根据 due_date（到期日）与当前日期自动计算
2. **实时更新**：每天定时任务更新回款计划状态，标识逾期计划
3. **包含状态**：逾期分析包括 PENDING、PARTIAL、OVERDUE 状态的计划

### 5.4 财务数据不可逆

1. **操作保护**：关键的财务操作（如回款确认、发票审批通过）一经完成，不允许普通角色修改或删除
2. **冲正机制**：仅支持有权限的财务人员执行冲正操作并记录日志
3. **审计追踪**：所有财务操作（确认、审批、核销）必须记录不可篡改的操作日志

## 6. 业务指标

### 6.1 关键指标

- **应收账款总额**：所有待回款金额的总和
- **逾期金额**：已逾期的应收账款金额
- **逾期率**：逾期金额 / 应收账款总额
- **回款确认率**：已确认回款金额 / 已登记回款金额
- **平均回款周期**：从合同签订到回款完成的平均天数
- **坏账风险率**：90天以上逾期金额 / 应收账款总额

### 6.2 监控指标

- **待确认回款数量**：需要财务确认的回款记录数量
- **平均确认时长**：从回款登记到财务确认的平均时间
- **发票审批通过率**：已批准发票 / 已提交发票
- **客户欠款排名**：按欠款金额和逾期天数的客户排名

## 7. 常见问题

### Q1: 财务确认回款后可以修改吗？

A: 财务确认后的回款记录不能由普通销售角色修改。如需修改，需要财务人员或有权限的管理员进行冲正操作，并记录操作日志。

### Q2: 如何处理回款金额与发票金额不符的情况？

A: 系统支持一个回款记录关联多个发票，但所有发票的总金额不能超过回款金额。如发票总金额超过回款金额，系统会拒绝确认。

### Q3: 账龄分析的数据是实时的吗？

A: 是的，账龄分析根据当前日期和回款计划的到期日实时计算逾期天数和金额。

### Q4: 逾期预警可以导出吗？

A: 可以。逾期预警接口支持分页查询，可以通过多次调用获取完整数据，然后导出为Excel或其他格式。

### Q5: 财务角色可以查看所有数据吗？

A: 是的。为了便于全局财务审核与审计，财务角色具有查看全公司所有客户、合同、回款计划及发票数据的权限。

## 8. 最佳实践

### 8.1 回款确认管理

1. **定期确认**：建议财务团队每日定时处理待确认回款，避免业务延误
2. **严格核对**：确认回款时必须核对银行实际到账信息
3. **及时关联发票**：确认回款时尽可能关联对应的发票申请，确保票款匹配

### 8.2 风险监控

1. **每日检查**：每天查看逾期预警列表，及时跟进高风险客户
2. **定期分析**：每周或每月进行账龄分析，评估整体坏账风险
3. **主动沟通**：对于逾期较长的客户，主动与销售团队沟通催收方案

### 8.3 数据管理

1. **准确记录**：确保所有财务确认操作都填写明确的备注说明
2. **定期审计**：定期检查操作日志，确保财务操作的合规性
3. **报表分析**：利用财务报表功能，为管理层提供决策支持
