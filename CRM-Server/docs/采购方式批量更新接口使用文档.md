## 📋 接口列表

### 1. 批量更新阶段模板

**接口地址：**
```
PUT /api/v1/procurement-methods/{method_id}/stages
```

**请求示例：**
```json
{
  "stages": [
    {
      "id": 9,
      "template_code": "PRE_QUAL",
      "stage_name": "资格预审（更新）",
      "win_probability": 25,
      "sort_order": 1,
      "is_default_start": 1,
      "can_skip": 0,
      "description": "第一阶段：提交资格预审文件"
    },
    {
      "id": 10,
      "template_code": "BID_EVAL",
      "stage_name": "投标评审（更新）",
      "win_probability": 55,
      "sort_order": 2,
      "is_default_start": 0,
      "can_skip": 0,
      "description": "第二阶段：技术标和商务标评审"
    },
    {
      "id": null,
      "template_code": "BID_OPEN",
      "stage_name": "开标评标",
      "win_probability": 75,
      "sort_order": 3,
      "is_default_start": 0,
      "can_skip": 0,
      "description": "第三阶段：公开开标并评标"
    }
  ]
}
```

**响应示例：**
```json
[
  {
    "id": 9,
    "procurement_method_id": 2,
    "template_code": "PRE_QUAL",
    "stage_name": "资格预审（更新）",
    "win_probability": 25,
    ...
  },
  {
    "id": 10,
    "procurement_method_id": 2,
    "template_code": "BID_EVAL",
    "stage_name": "投标评审（更新）",
    "win_probability": 55,
    ...
  },
  {
    "id": 18,
    "procurement_method_id": 2,
    "template_code": "BID_OPEN",
    "stage_name": "开标评标",
    "win_probability": 75,
    ...
  }
]
```

**字段说明：**
- `id` 有值 → 更新现有阶段
- `id` 为 null → 新增阶段
- `mark_delete`: true → 删除该阶段

### 2. 完整更新（方式 + 阶段）

**接口地址：**
```
PUT /api/v1/procurement-methods/{method_id}/full
```

**请求示例：**
```json
{
  "method": {
    "name": "公开招标（更新）",
    "description": "向所有潜在供应商发出的公开招标采购流程（已更新）",
    "is_active": 1,
    "sort_order": 2
  },
  "stages": [
    {
      "id": 9,
      "mark_delete": false
    },
    {
      "id": null,
      "template_code": "NEW_STAGE",
      "stage_name": "新增阶段"
    }
  ]
}
```

**响应示例：**
```json
{
  "id": 2,
  "code": "PUBLIC_BIDDING",
  "name": "公开招标（更新）",
  "description": "...",
  "is_active": 1,
  "sort_order": 2,
  "created_by": null,
  "updated_by": "ou_xxx",
  "created_time": "2026-02-13T03:14:03",
  "updated_time": "2026-02-13T11:30:00",
  "stage_templates": [...]
}
```

## ✅ 接口特性

### 原子性保证
- ✅ 使用数据库事务
- ✅ 要么全部成功，要么全部回滚
- ✅ 避免部分成功导致数据不一致

### 业务规则
- ✅ 自动检查默认起始阶段（每个采购方式只能有一个）
- ✅ 自动验证阶段编码唯一性
- ✅ 支持乐观锁机制防止并发冲突

### 错误处理
- ✅ 400 Bad Request：业务验证失败
- ✅ 401 Unauthorized：未登录
- ✅ 403 Forbidden：无权限
- ✅ 404 Not Found：采购方式不存在
- ✅ 500 Internal Server Error：服务器错误

## 📝 注意事项

1. **字段命名**：使用 `mark_delete` 而不是 `_delete`（Pydantic限制）
2. **ID判断**：有ID=更新，无ID=新增，`mark_delete=true`=删除
3. **权限控制**：使用 `require_permission()` 装饰器
4. **事务管理**：自动commit或rollback，前端无需处理
5. **乐观锁**：更新时自动处理 `version_lock` 字段
