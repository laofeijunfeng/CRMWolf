# 设计系统文档架构重构设计

**目标：** 将 `CRM-Docs/design-system/` 重构为可精确检索、每份 Markdown 不超过 100 行的模块化知识库，并用自动检查防止退化。

## 范围与原则

- 适用于目录下全部 Markdown，包括入口、规范、页面指南、迁移材料与治理说明。
- 不保留 `MASTER.md`、`ELEMENT-PLUS-MIGRATION.md` 或旧页面文件的兼容入口；更新所有内部引用。
- 一个规则只有一个权威来源（SSOT）；其它文档只概述用途并链接。
- 规则优先级固定为：页面特有规则 > 交互模式 > 组件规则 > 基础规范。同级冲突必须显式处理。
- 每个文件只负责一个可命名主题。超过 100 行时，按子主题拆分，不按编号机械切片。

## 信息架构

```text
design-system/
├── README.md
├── foundations/       # 色彩、排版、间距、图标、状态等设计令牌与基础规则
├── components/        # 表格、表单、弹窗、徽标等可复用组件的规范
├── patterns/          # 筛选、详情、审批、反馈等跨组件交互模式
├── pages/             # 列表、详情、表单、审批中心的组装与页面例外
├── migration/         # Element Plus 迁移策略、映射与例外
├── governance/        # 写作、边界、维护与检查说明
└── scripts/           # 文档质量检查脚本
```

各目录配置不超过 100 行的局部 `README.md`，列出该领域文件、适用场景和阅读顺序。正文文件采用主题名，而非序号或历史名称。

## 入口与索引

根 `README.md` 是唯一入口，且不超过 100 行，包含：

1. 快速阅读路径（按“新建页面”“修改组件”“执行迁移”“维护规范”）。
2. 全局优先级和 SSOT 规则。
3. 按任务/关键词组织的 Markdown 链接索引。
4. 各领域入口的简短说明。
5. 校验命令与贡献入口。

索引使用相对 Markdown 链接，不使用仅供人眼辨认的反引号路径。每份面向读者的规范必须由一个根或局部索引链接到；脚本、归档和生成物可排除，但须在检查配置中显式列出。

## 内容迁移与审阅

以当前 7 个文件为来源，按语义而不是原文件边界重新归类：

- `MASTER.md` 的通用视觉、组件和交互规则迁入 `foundations/`、`components/`、`patterns/`。
- 四份 `pages/*.md` 只保留页面构成、流程、页面特有约束和例外；通用规则改为链接。
- `ELEMENT-PLUS-MIGRATION.md` 拆为迁移总览、组件映射、风险/例外等主题文件。
- 现有 README 的操作性内容分流至权威主题，入口仅保留导航与规则。

迁移时建立审阅清单：

- **重复**：选定权威段落后合并，其它位置改为链接。
- **失效或过时**：修正不存在的文件路径、锚点和已废弃技术说明。
- **冲突**：记录来源、原文、影响范围与候选处理；任何会改变设计语义的冲突必须先取得用户确认。
- **遗漏**：将无归属但仍有效的规则分配到合适的领域文件。

## 文件格式与链接约定

每份说明文件开头使用一致的简短元数据区：用途、适用范围、权威性/优先级、关联文件。随后以任务导向标题组织内容，并在结尾提供“相关规范”链接。文本、示例与表格都计入 100 行；必要时优先拆分主题或将大示例提取为独立说明文件。

链接统一使用相对路径和 Markdown 锚点。页面指南只链接其依赖的权威规则，不复制其细节。重命名或移动后，仓库内不保留对旧路径的引用。

## 自动质量门禁

在 `scripts/` 中提供可本地运行并可接入现有 CI 的文档检查器。检查器接收设计系统根目录，失败时指出文件和原因，至少执行：

1. 每个纳入范围的 `.md` 文件行数不超过 100。
2. 相对本地 Markdown 链接的目标和锚点均存在。
3. 根或局部 README 可达全部面向读者的规范文件。
4. 旧文件名、旧路径和废弃引用不会重新出现。
5. 文档目录中的 Markdown 文件都拥有明确的索引归属。

检查范围与豁免以版本控制的配置明确声明；默认不允许豁免。为检查器提供覆盖超长文件、断链、未索引文件和旧路径引用的自动化测试。

## 验收标准

- `CRM-Docs/design-system/` 下所有说明 Markdown 均为 100 行或更少。
- 根入口和所有领域入口准确、可点击，并支持按任务定位规范。
- 每项规则有且仅有一个权威位置；页面说明无通用规则副本。
- 当前已知的遗漏索引和无效页面路径被清除。
- 重复内容已合并；任何语义冲突均有用户确认的处理结论。
- 文档检查器及其测试通过，可由开发者和 CI 运行。

## 非目标

- 不改变产品 UI、组件实现或迁移执行计划。
- 不维持历史文档路径兼容，也不提供重定向文件。
- 不对未识别为有效规则的历史内容做无依据扩写。
