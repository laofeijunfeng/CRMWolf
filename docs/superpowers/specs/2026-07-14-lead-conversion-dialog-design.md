# 线索 Sheet 内转化客户设计

## 目标

将线索转化统一为详情 Sheet 内的直接创建流程，避免跳转到独立转化页；删除失效的 `LeadConvert.vue` 和对应路由，从根源消除其对已删除 `@/utils/errorMessages` 模块的导入错误。

## 范围

- 在 `LeadDetailSheet.vue` 中实现客户转化 Dialog。
- 将 `Leads.vue` 的“转化为客户”入口统一到该 Sheet 流程。
- 删除独立转化页面及两条不再使用的路由。
- 不改动 `MyLeads.vue` 现有的独立旧转化弹窗。

## 交互流程

1. 用户在线索详情 Sheet 或线索列表中点击“转化为客户”。
2. 打开线索详情 Sheet；点击转化操作时，Sheet 保持打开并展示客户转化 Dialog。
3. Dialog 展示线索名称和以下客户信息：
   - 客户公司名称：必填，默认线索名称。
   - 所在城市：只读展示，继承线索城市；当前转化接口固定使用线索城市，因此不提供不会持久化的编辑入口。
   - 默认采购方式：必选，打开 Dialog 时加载选项。
   - 公司地址：选填。
4. 通过详情 Sheet 的“转化为客户”按钮进入时，打开转化 Dialog；通过线索列表的“转化为客户”菜单进入时，详情和采购方式加载完成后自动打开该 Dialog，避免用户重复确认转化意图。
5. 用户取消时只关闭 Dialog，不改变线索数据。未修改表单时允许 Esc、点击遮罩或关闭 Sheet 直接退出；已修改表单时，以上退出操作须先显示“放弃本次转化填写？”确认框。
6. 用户确认时先执行前端校验，再调用 `customerApi.convertLeadToCustomer`。字段按“客户公司名称（全宽）→ 所在城市 / 默认采购方式（宽屏两列、窄屏单列）→ 公司地址（全宽）”排列。
7. 提交期间禁用取消、确认、Esc、遮罩关闭和 Sheet 关闭；确认按钮显示加载状态，防止重复提交或用户失去操作上下文。
8. 成功后显示“线索已转化为客户”提示，关闭 Dialog 与 Sheet，向父级发出 `refresh`，并进入新建客户的详情页。
9. 失败时保留表单与 Sheet，使用 `handleApiError(error, '转化线索')` 提示原因，方便用户修改后重试。若接口返回“该线索已被转化”，则关闭 Dialog、刷新列表和 Sheet；若响应提供客户 ID，进入该客户详情页。

## 架构与数据流

- `LeadDetailSheet` 持有转化表单、脏数据状态、提交状态和采购方式选项。
- Sheet 打开并加载到线索详情后，转化按钮仅初始化/显示 Dialog；不立即路由跳转。
- 转化请求包含 `lead_id`、`account_name`、`address`、`default_procurement_method_id`。客户城市由后端从原线索 `city` 继承，因此 Dialog 仅只读展示该值，避免出现编辑但不保存的误导。
- `Leads.vue` 的转化菜单复用已有详情 Sheet 状态，设置选中线索 ID、打开 Sheet，并在数据加载完成后自动打开转化 Dialog；普通查看详情入口只打开 Sheet。
- Dialog 必须使用已有 shadcn-vue `Dialog`、`Input`、`Label`、`Select`、`Button` 和 `Skeleton`，图标统一使用 `lucide-vue-next`。样式仅使用 `variables-v2.scss` 的 `-v2` tokens，并保留组件库原生动态效果。
- `LeadConvert.vue` 不再被引用，因此删除该文件；路由表同步删除 `leads/:id/convert` 与 `leads/convert` 条目。

## 错误处理

- 采购方式选项加载中时，Select 显示加载状态；加载失败时显示统一 API 错误、字段附近的恢复说明和“重新加载”操作，并禁止用户提交，避免创建缺少必填采购方式的客户。
- 必填字段在失焦或提交时于字段下方显示错误并进入 Input error 状态；提交校验失败后焦点移至首个错误字段。
- 提交 API 失败时不关闭 Dialog、不清空表单；`handleApiError` 给出包含原因和恢复建议的标准化提示。
- 已转化线索不展示转化按钮，沿用当前状态条件。

## 验证

- 单元/类型检查覆盖删除失效路由后的编译与类型完整性。
- 通过前端构建确认不存在 `@/utils/errorMessages` 解析错误。
- 手动验收：通过详情 Sheet 入口打开 Dialog；通过列表转化入口时，Sheet 加载完成后自动打开 Dialog。填写名称、采购方式和地址并转化，观察防重复提交、成功提示、列表刷新和客户详情跳转。
- 验证未修改表单可直接退出、修改后关闭 Dialog/Sheet 会要求确认放弃；验证采购方式加载中、加载失败与重试状态；验证 API 失败后 Dialog 与填写内容保留。
- 验证必填字段错误在相邻字段下方展示、焦点移动到首个错误字段；在窄屏下表单为单列且无横向滚动，所有操作目标不少于 44px。
- 验证浅色和深色主题中的 Dialog、输入框、边框、禁用态与焦点态均清晰可辨；验证并发转化返回过期状态时，页面会刷新到最新数据。
